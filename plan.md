优先覆盖：

- 科举考试
- 朝堂斗争
- 多线并行
- 阶段收束
- 状态回填压力

### 2. 测试方法还需要标准化

后续每轮测试应固定产出：

- `input.md`
- `outline.md`
- `drafts.md`
- `states.md`
- `review.md`
- `decision.md`

并使用 `test/test-run-template.md` 统一评分和问题归因。

### 3. 工作流优化要避免过拟合

不再因为单个样本失败立刻修改 `workflow`。只有满足以下条件之一，才修改工作流：

- 同类失败连续出现 2 次
- 不同测试类型反复出现同一结构性问题
- 失败原因明确来自 Prompt 缺少硬约束
- 状态回填无法支撑下一批章纲，且不是样本输入问题
