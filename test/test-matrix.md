# 修仙搞笑长篇工作流测试矩阵

## 目标

验证工作流和后续执行器能否稳定生产修仙成长与情境喜剧并存的长篇正文。

所有新测试使用 `X` 前缀。已有历史运行可以保留，但不代表当前专项版本已通过。

## 测试原则

- 每轮优先测试一个主要风险。
- 先单章，再 3～5 章，最后 10～20 章。
- 必须保存真实正文和机械长度结果。
- 任一硬失败都不能被总分抵消。
- 测试具体设定只放在对应 `matrix-runs/` 目录。

## 核心指标

- 长度合格率
- 摘要化章节数
- 修炼体系矛盾数
- 资源连续性错误数
- 无来源状态事实数
- 知识与误会矛盾数
- 喜剧机制重复数
- 严肃后果被取消次数
- 需返工章节数
- 平均重试次数
- 第二批账本启动成功率

## 通用硬失败

- 未通过长度闸门仍进入状态回填。
- 失败章节之后继续提交正式章节。
- 修炼体系出现硬矛盾。
- 状态记录正文未发生内容。
- 资源凭空出现或消失。
- 主要情节依赖全员降智。
- 搞笑取消重大后果。
- 连续两章只有铺垫。
- 使用摘要稿冒充长正文测试。

## 工作流修改触发条件

只有满足以下条件之一才修改工作流：

1. 同一测试类型连续两轮出现同类失败。
2. 不同测试重复暴露同一结构问题。
3. Prompt 缺少明确约束。
4. 状态或账本无法支撑后续批次。
5. 喜剧疲劳规则无法阻断连续重复。
6. 修炼状态字段无法表达实际变化。
