# 测试计划与结果

## 测试对象

- `workflow/03-chapters.md`
- `workflow/04-draft.md`

## 测试目的

确认更新后的 Prompt 是否能把正文稳定推向：

- 更快进入冲突
- 更少说明文和空转心理
- 每章有卖点、有兑现点
- 章末钩子具体
- 多线或多目标章节不写平

## 测试场景

### 1. 首章开局章

- 测试内容：
  - 主角亮相速度
  - 当前困境落地速度
  - 首章卖点是否够快
  - 是否存在“只有钩子没有兑现”
- 对应产物：
  - `drafts/ch01-v1.md`
  - `drafts/ch01-v2.md`
  - `drafts/ch01-v3.md`
  - `drafts/ch01-v4.md`
- 结果：
  - `v1` 偏慢，解释偏多
  - `v2` 明显提速
  - `v3` 增加了主角差异动作
  - `v4` 基本达到可用

### 2. 首章后承接章

- 测试内容：
  - 首章后的承接会不会掉速
  - 专业感会不会滑回说明文
  - 是否仍有兑现点
- 对应产物：
  - `drafts/ch02-verify.md`
- 结果：
  - 通过
  - 说明更新后的规则不只对首章有效

### 3. 施压章

- 测试内容：
  - 施压是否具体
  - 会不会整章只压不兑
  - 章末是否能明确推下去
- 对应产物：
  - `drafts/pressure-chapter.md`
- 结果：
  - 通过
  - 施压与兑现可以共存

### 4. 诗词破圈章

- 测试内容：
  - 会不会写成装逼流水账
  - 会不会写成“别人集体震惊”的套路章
  - 诗词是否真正服务情景
- 对应产物：
  - `drafts/poem-breakout.md`
- 结果：
  - 通过
  - 爽点兑现明确

### 5. 多目标推进章

- 测试内容：
  - 主线推进、关系变化、伏笔、钩子是否会互相打架
  - 是否容易写成信息说明会
- 对应产物：
  - `drafts/multi-goal-chapter.md`
- 结果：
  - 通过
  - 多目标章节未明显写平

### 6. 轻情绪/关系推进章

- 测试内容：
  - 没有强对抗时，关系变化是否仍有推进
  - 会不会写成软、平、散
- 对应产物：
  - `drafts/emotion-relationship.md`
- 结果：
  - 通过
  - 父子关系松动形成了明确兑现点

### 7. 连续 3 章组测

- 测试内容：
  - 连续阅读是否掉速
  - 每章是否都有独立兑现点和承接钩子
- 对应产物：
  - `drafts/batch-3chapters.md`
- 结果：
  - 通过
  - 三章分别完成试犁、口碑人情、旧账反查

### 8. 失败/受挫章

- 测试内容：
  - 主角没有赢透时，读者是否仍有追读理由
  - 失败是否能转化为下一步问题定位
- 对应产物：
  - `drafts/failure-setback.md`
- 结果：
  - 通过
  - 当众失败后仍保留了二次改良和复试钩子

### 9. 多线正式交汇章

- 测试内容：
  - 农具线、诗名线、地方势力线是否能在同一事件中交汇
  - 多线是否混乱
- 对应产物：
  - `drafts/multi-line-merge.md`
- 结果：
  - 通过
  - 梁存义到场让诗名压力转为现场考察

### 10. 朝堂/权谋试探章

- 测试内容：
  - 信息密、立场多、陷阱多时是否能写清
  - 是否避免“主角开天眼骂翻全场”
- 对应产物：
  - `drafts/court-politics.md`
- 结果：
  - 通过
  - 权谋试探有具体议题和利益结构

### 11. 长信息揭示章

- 测试内容：
  - 大量背景进入时，是否会变成说明会
  - 信息揭示后是否改变主角目标
- 对应产物：
  - `drafts/long-info-reveal.md`
- 结果：
  - 通过
  - 通过县志、旧图、问答逐步揭示地方旧网

### 12. 状态回填联动测试

- 测试内容：
  - 正文写完后，状态记录是否能反哺下一章
  - 回填是否记录人物、关系、资源、伏笔变化
- 对应产物：
  - `drafts/state-feedback-loop.md`
- 结果：
  - 通过
  - 下一章前置状态明确

### 13. 同一卖点疲劳测试

- 测试内容：
  - 连续使用“种地/改良”卖点时是否疲劳
  - 是否能通过更换兑现角度保持新鲜感
- 对应产物：
  - `drafts/motif-fatigue.md`
- 结果：
  - 通过
  - 需要每次更换具体爽点：省力、均匀、水路、成本、可复制
