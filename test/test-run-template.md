## 5. 正文质量检查

### 场景与推进

- [ ] 正文不是摘要稿。
- [ ] 核心场景得到展开。
- [ ] 每章有真实状态变化。
- [ ] 本章兑现清楚。
- [ ] 章末有具体行动方向。

### 修仙一致性

- [ ] 境界和能力边界一致。
- [ ] 没有临时创造能力救场。
- [ ] 突破有前置、过程、资源和代价。
- [ ] 资源有来源、数量和去向。
- [ ] 执行器计算后的资源余额与状态声明一致。
- [ ] 伤势和限制得到继承。
- [ ] 战斗或冲突结果符合现场条件。

### 搞笑质量

- [ ] 笑点来自人物、信息差、规则或行动。
- [ ] 没有全员降智。
- [ ] 没有大量网络梗堆砌。
- [ ] 没有依赖旁白解释笑点。
- [ ] 喜剧没有取消严肃后果。
- [ ] 核心角色语言和反应有区别。
- [ ] 主要喜剧机制没有连续疲劳。
- [ ] 回调产生了新信息或新后果。

| 维度 | 0 分 | 1 分 | 2 分 | 得分 |
