# 第106~110章批次评审（v1）

## 一、总体判断

第106~110章把 `101~105` 留下的送递门槛，继续推进到了“留人试用 -> 看人问话 -> 压稿重递 -> 局内见面”这一层。

这一批最关键的结果是：

1. 留人不是口头结果，而是实场景
2. 看人不再只是判断句，而是问话过程
3. 改口不是总结，而是具体删改动作
4. 最后成功把阶段入口推进到局内见面

## 二、单章结果

- 第106章：留人试压成立
- 第107章：只看人不看稿成立
- 第108章：改口压稿成立
- 第109章：半接纳状态成立
- 第110章：局内见面成立

## 三、本批主要问题

### 1. 第107章和第108章仍是最高风险

这两章最容易退成纯问答和纯修稿。当前通过“多轮问话”和“明确删改动作”暂时稳住了。

### 2. 第110章是阶段门，不是阶段终点

本章只负责把主角带进更深层，不承担翻盘。这样更符合当前批次节奏。

## 四、对工作流的修订结论

1. **03-chapters**
   - 试用章必须把“留人”拆成实景
   - 问话章必须带具体判断点

2. **04-draft**
   - 改口章要写动作，不写空泛体会
   - 局内见面章必须有明确门槛句

3. **05-update-state**
   - 状态回填要继续抓“下一章前置状态”
   - 对“更深入口”要单独记录

## 五、下一批约束

如果继续往下写，重点继续观察：

1. 局内之后会不会掉成空谈
2. 写人还是写路能否继续撑住动作量
3. 更高层压力会不会让正文过度解释
