# 新模具复跑细纲 v1：第106~110章

> 用途：承接第105章“人先留下，稿子再说”的结果，继续验证留人试用、单独问话、改口递案和更深层试压是否能连续成立。

## 批次目标

- 验证“人先留下”之后，主角是否真的进入了更细的观察和试用
- 验证“只看人，不看稿”会不会把正文写成纯对话
- 验证改口后的新稿是否能继续保住送递感

## 批次风险点

- 留人章容易写成被动等待
- 问话章容易写成纯聊天
- 改口章容易写成纯修辞

---

## 第106章：先留人
- **功能**：争
- **目标字数**：2000~2500字
- **本章即时卖点**：底稿暂时不看，先把人留下，看他到底是不是那支笔
- **本章兑现点**：让主角进入更深一层的单独试用场
- **最晚冲突落点**：前30%
- **核心事件**：王兆陪被留下单独候问，底稿暂不被接收，只被安排在旁，先看他本人
- **冲突类型**：留人试压 / 单独观察
- **人物发展**：主角意识到自己已经不只是递稿的人，而是被试的人
- **信息释放**：对方更在意他的判断和气口，不急着看全文
- **钩子**：有人轻声说：“稿子先放着，先看看你这人能不能留下。”
- **本章不可删场景**：单独候问必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：105章送出后，主角得到“人先留下”的回应
  - 必做事件：
    1. 把“留下”写成实景，不是通知
    2. 让主角感到自己被单独观察
    3. 留下一个后续问话入口
  - 禁止越界：
    1. 不直接进入正式接稿
    2. 不让常先生提前现身
  - 结束状态：主角进入更深一层试用
  - 下章承接：围绕“只看人不看稿”继续试压
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 留下与安置，负责把人先留下的状态落地
    2. 候问与观察，负责拉紧试压
    3. 留言收尾，负责导向下一章问话
  - 字数支撑理由：本章是试用起点，天然需要环境、等待和暗压三段
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约800~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。留人本身就是事件。

## 第107章：只看人，不看稿
- **功能**：争 / 压
- **目标字数**：2000~2500字
- **本章即时卖点**：对方不先看纸，只拿王兆陪本人做试题
- **本章兑现点**：把“看人”的标准写成具体问话和现场反应
- **最晚冲突落点**：前35%
- **核心事件**：主角接受一轮只看人不看稿的问话，内容围绕仓务、口径、救人与写证的次序展开
- **冲突类型**：试用问话 / 口径试压
- **人物发展**：主角开始知道，对方要的不是能写，而是能不能稳
- **信息释放**：真正决定去留的，是他答问时的轻重、迟快和分寸
- **钩子**：问话人只回一句：“人倒是能看，稿子还得再压。”
- **本章不可删场景**：至少一轮连续问答必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角被留下，进入试问
  - 必做事件：
    1. 让对方只看人，不看稿
    2. 把问话写成多轮，而不是一句带过
    3. 让主角第一次意识到“稳”比“会写”更值钱
  - 禁止越界：
    1. 不直接接收底稿
    2. 不提前露出最终站队
  - 结束状态：主角通过或勉强通过第一轮人审
  - 下章承接：围绕口径进入改写
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 开场观察，负责建立试问气氛
    2. 多轮问答，负责把“看人”写实
    3. 结果落点，负责导向压稿
  - 字数支撑理由：本章是问答章，但问答必须有事件推进和判断变化
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约850~1050字
    3. 场景3：约450~650字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。问话本身包含分辨和压迫。

## 第108章：改口
- **功能**：压
- **目标字数**：2000~2500字
- **本章即时卖点**：不只是改稿，而是改到对方愿意再看一遍
- **本章兑现点**：把“口径”从写法层面推到送递层面
- **最晚冲突落点**：前30%
- **核心事件**：王兆陪根据前章反馈重修说法，把太硬、太满、太急的地方压下去，只留真正能递的骨头
- **冲突类型**：改口 / 反向压稿
- **人物发展**：主角开始意识到，真正的难不是写出来，而是写到别人愿意接
- **信息释放**：对方不反感内容，反感的是内容没留门
- **钩子**：改到最后，连他自己都觉得这份稿子像另一只手写的
- **本章不可删场景**：必须出现明确改口动作和删改结果
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：前一轮试问后，主角收到压稿反馈
  - 必做事件：
    1. 明确哪些口径要改
    2. 把改口落成具体删改
    3. 让主角感到自己的写法变了
  - 禁止越界：
    1. 不把改口写成抽象感悟
    2. 不直接完成最终接收
  - 结束状态：主角拿到可再递版本
  - 下章承接：围绕“再看一次”进入更深层观察
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 反馈拆解，负责定改口方向
    2. 删改重写，负责把压稿动作写实
    3. 重立版本，负责给下一轮递送铺路
  - 字数支撑理由：本章不是总结修稿，而是有明确删改动作的剧情
  - 场景级字数预算：
    1. 场景1：约550~700字
    2. 场景2：约850~1000字
    3. 场景3：约500~650字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。改口就是行动。

## 第109章：再看一遍
- **功能**：破
- **目标字数**：2000~2500字
- **本章即时卖点**：稿子压过之后，对方才真正重新看一遍
- **本章兑现点**：让主角被更明确地纳入下一层视野
- **最晚冲突落点**：前35%
- **核心事件**：主角带着改口版再次受看，对方不再只看字面，而是开始看他能不能扛事
- **冲突类型**：再审 / 更深层试探
- **人物发展**：主角开始从“候选笔”向“可用的人”推进
- **信息释放**：对方愿意再看一遍，说明第一轮试压已经过了大半
- **钩子**：对方只说：“可以先放进来，但还不算你的人。”
- **本章不可删场景**：再次受看必须写出和前一轮的差别
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角拿着改口后的版本回到试用场
  - 必做事件：
    1. 让对方重新看稿
    2. 写出与前轮不同的判断
    3. 留下半接纳状态
  - 禁止越界：
    1. 不直接完全通过
    2. 不提前让常先生坐实
  - 结束状态：主角被允许继续往里走一点
  - 下章承接：进入真正的局内接触
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 重看前的等待，负责拉试压
    2. 再审与判断，负责写出差别
    3. 半接纳结果，负责导向下一章
  - 字数支撑理由：再审章必须有前后差异，否则会空
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约800~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。再审本身就是剧情。

## 第110章：局内见面
- **功能**：得
- **目标字数**：2000~2500字
- **本章即时卖点**：留下、压稿、再看之后，主角终于被带进局内见到真正说得上话的人
- **本章兑现点**：把前几章的试用和压稿转成局内见面
- **最晚冲突落点**：前30%
- **核心事件**：王兆陪被带入更深一层见面场合，见到真正能定下一步的人，得到下一阶段的明确门槛
- **冲突类型**：局内见面 / 阶段进入
- **人物发展**：主角正式从外线笔，走进局内写手候位
- **信息释放**：常先生的影子终于变成能说话的现实压力
- **钩子**：对方只问一句：“你是来写人，还是来写路？”
- **本章不可删场景**：局内见面必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角已通过前几轮试压
  - 必做事件：
    1. 进入更深一层见面场
    2. 写出真正说得上话的人
    3. 落下下一阶段的明确门槛
  - 禁止越界：
    1. 不直接翻盘
    2. 不把局内见面写成散聊
  - 结束状态：主角正式进入更深一层局内
  - 下章承接：围绕“写人还是写路”进入新阶段
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 入场与见面前压场，负责拉层级
    2. 正式见面，负责兑现前批结果
    3. 问题抛出，负责给下一阶段钩子
  - 字数支撑理由：本章是阶段进入章，不是结论章
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约800~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。层级变化足够支撑成稿。

## 本批次备注

- **批次风险点**：第107章和第108章最容易写成纯问答或纯修稿，正文试写时应优先盯住动作和反馈
