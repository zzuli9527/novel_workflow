# 新模具复跑细纲 v1：第116~120章

> 用途：承接第115章常先生明面召见，验证高层对谈、调度权线和鲁伙计现实危机是否能继续保持场景推进。

## 批次目标

- 验证常先生明面出场是否还能保持实场景
- 验证调度权线是否能落到现实入口
- 验证鲁伙计线是否继续和高层压力绑定

## 批次风险点

- 常先生章容易写成解说
- 调度权容易变成抽象概念
- 鲁伙计线如果断开，现实危机会被削弱

---

## 第116章：明日来见我
- **功能**：压
- **目标字数**：2000~2500字
- **本章即时卖点**：常先生终于明面召见，但不是给答案，而是先看王兆陪敢把笔落到哪
- **本章兑现点**：常先生明面出场，并把“调度权”压成新的考题
- **最晚冲突落点**：前30%
- **核心事件**：王兆陪带着调度权判断去见常先生，常先生不直接否定，只问他凭什么把旧院、残纸、鲁伙计并成一条线
- **冲突类型**：明面召见 / 高层试压
- **人物发展**：主角第一次在常先生面前完整说出调度权判断
- **信息释放**：常先生真正关注的是主角能不能把零散事写成可用路数
- **钩子**：常先生让他明日去看一份“不该存在的出入册”
- **本章不可删场景**：常先生明面召见必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角收到常先生召见
  - 必做事件：
    1. 写出见面前的压力
    2. 常先生明面问调度权
    3. 留下出入册入口
  - 禁止越界：
    1. 不直接给出幕后主使
    2. 不直接解决鲁伙计线
  - 结束状态：主角拿到下一步实物入口
  - 下章承接：围绕出入册进入核验
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 赴约与入场，负责拉常先生明面压力
    2. 面谈试压，负责兑现调度权判断
    3. 出入册入口，负责把对谈落回实物
  - 字数支撑理由：本章不是纯谈话，有赴约、面谈、实物入口三段
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约850~1050字
    3. 场景3：约450~650字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。常先生明面试压足够支撑正文。

## 第117章：不该存在的出入册
- **功能**：争
- **目标字数**：2000~2500字
- **本章即时卖点**：出入册上没有鲁伙计的名字，却处处有他留下的影子
- **本章兑现点**：把调度权从判断推进到可查证的账册痕迹
- **最晚冲突落点**：前30%
- **核心事件**：王兆陪核验出入册，发现旧院中转记录被改过，但改口留下了时间断层
- **冲突类型**：实证核验 / 账册断层
- **人物发展**：主角开始用调度权思维读账册
- **信息释放**：真正被藏的不是鲁伙计名字，而是能调动旧院的时辰和批次
- **钩子**：一处被补写的时辰，正好卡在鲁伙计失踪后的半个时辰
- **本章不可删场景**：核验出入册必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角拿到出入册入口
  - 必做事件：
    1. 现场核验出入册
    2. 找到被改过的时间断层
    3. 把断层和鲁伙计失踪时间对上
  - 禁止越界：
    1. 不直接找到鲁伙计
    2. 不直接拿到完整调度名单
  - 结束状态：主角确认调度权有账册痕迹
  - 下章承接：围绕被补写时辰继续追
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 看册前的规矩，负责立核验场
    2. 查册与对时，负责找断层
    3. 时辰钩子，负责接鲁伙计线
  - 字数支撑理由：本章是实证核验，不靠心理解释
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约850~1050字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。账册核验可完整展开。

## 第118章：半个时辰
- **功能**：争 / 压
- **目标字数**：2000~2500字
- **本章即时卖点**：半个时辰的断层，可能就是鲁伙计活着被转走的最后窗口
- **本章兑现点**：把账册断层推进成现实追索窗口
- **最晚冲突落点**：前25%
- **核心事件**：主角沿半个时辰追查旧院出车和人手，确认有一辆不该走的空车在那段时间进出
- **冲突类型**：时限追索 / 车马线索
- **人物发展**：主角把写路能力转成现实追索能力
- **信息释放**：出入册改口真正遮的是一辆空车
- **钩子**：车辙尽头指向一处临时税栈
- **本章不可删场景**：半个时辰内的车马追索必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角确认出入册有半个时辰断层
  - 必做事件：
    1. 对时并追查车马
    2. 确认空车进出
    3. 将线索引向临时税栈
  - 禁止越界：
    1. 不直接救出鲁伙计
    2. 不直接确认空车里的人
  - 结束状态：主角将调度权线推进到税栈
  - 下章承接：进入临时税栈探查
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 对时复核，负责确立窗口
    2. 车马追查，负责现实动作
    3. 税栈指向，负责新地点钩子
  - 字数支撑理由：本章是行动追索章，场景天然充足
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约850~1050字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。车马追索支撑体量。

## 第119章：临时税栈
- **功能**：破
- **目标字数**：2000~2500字
- **本章即时卖点**：税栈不是关人的地方，却是把活人变成账面空格的地方
- **本章兑现点**：让税栈成为调度权线的现实节点
- **最晚冲突落点**：前30%
- **核心事件**：王兆陪和邢管事探查临时税栈，发现这里负责替不该出现的人和物做短暂停格
- **冲突类型**：节点探查 / 账面遮掩
- **人物发展**：主角意识到调度权不是一人发令，而是一串节点配合
- **信息释放**：税栈能把实际转运伪装成正常滞留
- **钩子**：税栈墙后听见有人咳了一声，像鲁伙计
- **本章不可删场景**：税栈探查必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角沿车辙追到临时税栈
  - 必做事件：
    1. 探查税栈规则
    2. 发现账面遮掩方式
    3. 听到疑似鲁伙计的咳声
  - 禁止越界：
    1. 不直接救人
    2. 不直接暴露身份硬闯
  - 结束状态：主角确认人可能近在眼前
  - 下章承接：围绕是否冒险确认人声进入取舍
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 税栈外观察，负责立地点
    2. 入内试探，负责写遮掩规则
    3. 咳声钩子，负责拉现实危机
  - 字数支撑理由：税栈探查有清晰场景和压力
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约800~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。现实探查足够成稿。

## 第120章：不能硬闯
- **功能**：压 / 得
- **目标字数**：2000~2500字
- **本章即时卖点**：人可能就在墙后，可王兆陪第一次必须忍住不救
- **本章兑现点**：让主角在救人冲动和写路策略之间做一次更成熟的选择
- **最晚冲突落点**：前30%
- **核心事件**：王兆陪确认墙后可能有人，却没有硬闯，而是用税栈规则反写一条能逼对方转移的路
- **冲突类型**：救人与布局 / 成熟取舍
- **人物发展**：主角从“急着救人”成长为“先让对方不得不动”
- **信息释放**：税栈一旦被写成节点，对方就不能继续稳藏
- **钩子**：常先生看完新稿，只说：“这一次，你开始会等了。”
- **本章不可删场景**：主角忍住不硬闯的选择必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角听见疑似鲁伙计咳声
  - 必做事件：
    1. 确认不能硬闯
    2. 反写税栈节点
    3. 得到常先生“会等”的反馈
  - 禁止越界：
    1. 不直接救回鲁伙计
    2. 不直接解决税栈线
  - 结束状态：主角学会用写路逼对方动
  - 下章承接：等待对方转移或反扑
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 墙后人声与冲动，负责拉情绪压力
    2. 克制与反写，负责落成熟选择
    3. 常先生反馈，负责阶段收束
  - 字数支撑理由：本章有情绪、策略、反馈三段，不是总结章
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约850~1050字
    3. 场景3：约450~650字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。取舍和反写动作明确。

## 本批次备注
- **批次范围**：第116~120章
- **批次目标**：验证常先生明面召见、出入册核验、税栈节点和成熟取舍是否能连续成立
- **批次风险点**：第116章和第120章容易写成高层点评，正文试写时必须落到出入册和税栈规则
- **对下一批的约束**：如果常先生反馈过重，下批应减少解说，增加对方反扑和现实动作
