# 新模具复跑细纲 v1：第126~130章

> 用途：承接第125章半截篓绳和鲁伙计结法，验证篓绳追踪、堆工暗线风险、可救窗口和救人与保线冲突是否能连续成立。

## 批次目标

- 验证篓绳结法能否指向真正藏人点
- 验证堆工递线是否带来新风险
- 验证鲁伙计是否从“可能活着”推进到“可救但有代价”

## 批次风险点

- 回到单纯救人线
- 堆工递线太顺
- 鲁伙计无成本获救，削弱前面压力

---

## 第126章：篓绳结法
- **功能**：争
- **目标字数**：2000~2500字
- **本章即时卖点**：半截篓绳上的结法，只有真正常搬货的人才会打
- **本章兑现点**：篓绳不再只是信物，而是指向具体搬运口
- **最晚冲突落点**：前30%
- **核心事件**：王兆陪和邢管事拆看篓绳结法，确认它不是普通篓绳，而是鲁伙计常用的“反扣短结”
- **冲突类型**：物证识别 / 搬运口追踪
- **人物发展**：主角从保住证据转向沿证据追人
- **信息释放**：鲁伙计可能在被迫搬运或被藏入搬运链时留下结法
- **钩子**：这种结法只在西河渡下游一处小平码头常见
- **本章不可删场景**：拆解篓绳结法必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角拿到半截篓绳
  - 必做事件：
    1. 拆解结法
    2. 确认与鲁伙计习惯相合
    3. 指向下游小平码头
  - 禁止越界：
    1. 不直接找到鲁伙计
    2. 不直接确认堆工身份
  - 结束状态：主角获得真正藏人点方向
  - 下章承接：围绕小平码头探查
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 篓绳拆看，负责立物证
    2. 结法比对，负责确认鲁伙计关系
    3. 小平码头指向，负责给下一章入口
  - 字数支撑理由：本章是物证识别章，动作和判断明确
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约750~950字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。篓绳结法能支撑完整比对。

## 第127章：小平码头
- **功能**：争
- **目标字数**：2000~2500字
- **本章即时卖点**：小平码头不起眼，却正好能把篓、车、人三条线接起来
- **本章兑现点**：找到真正藏人链条的现实地点
- **最晚冲突落点**：前25%
- **核心事件**：主角和邢管事暗查小平码头，发现这里专门接短停篓货和无名脚夫
- **冲突类型**：藏点探查 / 短停链条
- **人物发展**：主角逼近鲁伙计所在链条
- **信息释放**：鲁伙计未必被关在一处，而是在短停链条里被挪动
- **钩子**：码头账牌上出现一个被划掉的“鲁”字残痕
- **本章不可删场景**：小平码头探查必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角沿篓绳结法追到小平码头
  - 必做事件：
    1. 探查小平码头
    2. 确认短停链条
    3. 找到被划掉的“鲁”字残痕
  - 禁止越界：
    1. 不直接救回鲁伙计
    2. 不直接暴露主角身份
  - 结束状态：主角确认已经逼近鲁伙计链条
  - 下章承接：围绕堆工递线风险继续查
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 码头外围观察，负责立地点
    2. 短停链条确认，负责写规则
    3. “鲁”字残痕，负责强钩子
  - 字数支撑理由：本章是现实地点探查，动作充足
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约800~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。地点和规则都能展开。

## 第128章：递线的人也会被盯上
- **功能**：压
- **目标字数**：2000~2500字
- **本章即时卖点**：堆工递出半截篓绳后，自己也开始消失
- **本章兑现点**：证明递线不是无成本行为
- **最晚冲突落点**：前30%
- **核心事件**：主角回查递线堆工，发现他被调离堆场，并留下另一个更短的警告
- **冲突类型**：暗线受压 / 新风险
- **人物发展**：主角意识到帮助自己的人也会被收口
- **信息释放**：对方已经察觉有人漏线
- **钩子**：堆工留下“别去明泊棚”四个字，反而指向一个更危险地点
- **本章不可删场景**：堆工受压必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角确认小平码头有鲁伙计残痕
  - 必做事件：
    1. 回查递线堆工
    2. 发现堆工被调离或失踪
    3. 留下“别去明泊棚”警告
  - 禁止越界：
    1. 不直接找到堆工本人
    2. 不直接确认明泊棚真相
  - 结束状态：主角知道明泊棚极危险但必须查
  - 下章承接：围绕明泊棚进入可救窗口
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 回查堆工，负责承接递线代价
    2. 调离痕迹，负责证明对方收口
    3. 警告入手，负责导向明泊棚
  - 字数支撑理由：本章是暗线受压章，冲突明确
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约800~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。堆工受压能构成完整事件。

## 第129章：明泊棚里有人
- **功能**：破
- **目标字数**：2000~2500字
- **本章即时卖点**：警告说别去，王兆陪偏偏从警告里看见了人还活着的可能
- **本章兑现点**：鲁伙计从线索人物推进到可救目标
- **最晚冲突落点**：前25%
- **核心事件**：主角远探明泊棚，听见里面有人用鲁伙计旧口音低声报数
- **冲突类型**：可救窗口 / 高危确认
- **人物发展**：主角第一次确认鲁伙计极可能还活着
- **信息释放**：明泊棚不是长期关押地，而是转移前的临时停口
- **钩子**：明泊棚天亮前要转一批“空篓”
- **本章不可删场景**：明泊棚远探和人声确认必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角拿到“别去明泊棚”的警告
  - 必做事件：
    1. 远探明泊棚
    2. 听见疑似鲁伙计人声
    3. 得知天亮前要转“空篓”
  - 禁止越界：
    1. 不直接救人
    2. 不直接硬闯棚内
  - 结束状态：主角确认可救窗口存在
  - 下章承接：围绕救人与保线再次取舍
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 明泊棚外围，负责拉高危险
    2. 人声确认，负责兑现鲁伙计可能活着
    3. 转篓消息，负责给下一章时限
  - 字数支撑理由：本章是高危确认章，场景张力足够
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约800~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。人声确认足够支撑正文。

## 第130章：救人会断线
- **功能**：压 / 得
- **目标字数**：2000~2500字
- **本章即时卖点**：鲁伙计可能就在眼前，可现在救人会让整条调度权线断掉
- **本章兑现点**：再次逼主角在救人和保线之间做成熟选择
- **最晚冲突落点**：前30%
- **核心事件**：主角判断明泊棚可救但不能硬救，决定先让邢管事布一条后手，同时把明泊棚写进新稿，逼对方转移时露更多口
- **冲突类型**：救人与保线 / 成熟布局
- **人物发展**：主角从“会等”推进到“会设后手”
- **信息释放**：救鲁伙计需要抓住转移时机，而不是抢眼前一刻
- **钩子**：邢管事带回一句：“若天亮前不动，人可能就不在棚里了。”
- **本章不可删场景**：救人与保线的取舍必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角确认明泊棚有可救窗口
  - 必做事件：
    1. 判断硬救代价
    2. 布置后手
    3. 用天亮前时限收尾
  - 禁止越界：
    1. 不直接救回鲁伙计
    2. 不直接断掉调度权线
  - 结束状态：主角进入救人前最后布局
  - 下章承接：天亮前转篓行动
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 硬救代价判断，负责拉冲突
    2. 后手布置，负责给主角主动动作
    3. 天亮前时限，负责下一章钩子
  - 字数支撑理由：本章是取舍和布局章，有明确动作
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约850~1050字
    3. 场景3：约450~650字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。取舍和后手都能展开。
