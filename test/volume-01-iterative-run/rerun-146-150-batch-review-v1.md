# 第146~150章批次评审（v1）

## 一、总体判断

第146~150章把第145章废稿里的“王兆陪私匿……”推进成了正式文书反咬，并让反制从“发现废稿”升级为“递送路径、红泥拖痕、值房案板、残墨纸屑”的实物证据链。

这一批的关键验证点基本成立：

1. 正式告文让王兆陪处境变窄
2. 鲁伙计没有公开，仍是隐藏压力
3. 反制没有靠口才，而是靠递送簿、墨色、红泥、案板残证
4. 书办值房没有立刻降智暴露，而是通过扣邢管事反压证据来源

## 二、单章结果

- 第146章：正式告文反咬成立
- 第147章：递送路径漏洞成立
- 第148章：三稿红泥并线成立
- 第149章：邢管事被扣和证据来源风险成立
- 第150章：旧案板残证与不能公开的权衡成立

## 三、本批主要问题

### 1. 文书证据线仍有“查档案化”风险

第147~148章都依赖文书细节，如果正文扩成 2000~2500 字，必须增加人的反应、递送小吏的闪避、邢管事的担忧，否则会像流程说明。

### 2. 第150章的“废库堵门”需要防止套路化

堵门钩子成立，但下一批不能简单写成对方抓现行。更好的方向是让堵门者带来新的交易、误导或第二层陷阱。

### 3. 证据不能公开这一点有效，但需要在工作流中继续强调

当前批次验证出一个重要规则：拿到证据不等于可以立刻使用。正文写作阶段必须强制检查“公开代价”，否则长篇连续写作容易过早翻盘。

## 四、对工作流的修订结论

需要对 `workflow/04-draft.md` 做一个小补丁：

- 在正文写作自检里加入“证据公开代价检查”
- 约束主角拿到证据后，必须判断公开会牵连谁、暴露哪条隐藏线、导致什么反扑
- 防止证据一出现就立刻翻盘，破坏连续追读压力

`workflow/03-chapters.md` 暂不需要修改。当前章纲已经能通过“禁止越界”和“结束状态”控制不能立刻翻盘。

`workflow/05-update-state.md` 暂不需要修改。状态回填已经能记录“证据资源”和“风险资源”。

## 五、下一批约束

如果继续往下写，重点观察：

1. 废库堵门者不能只是来抓人，必须带出新的压力或误导
2. 王兆陪要保住旧案板残证，同时不能公开鲁伙计
3. 邢管事仍未脱困，不能轻易恢复工具人功能
4. 正式文书反咬不能一批内解决，至少继续压住主角 3~5 章
