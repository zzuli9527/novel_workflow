# 新模具复跑细纲 v1：第161~165章

> 用途：承接第160章“同谋书生”空栏，验证新告文盖印前流程、邢管事替罪风险、王兆陪被写入同谋栏的逼近，以及批次级“对手认知账本”是否能承接长期误判链。

## 批次目标

- 验证新告文进入递签、核名、请印或预盖印流程
- 验证邢管事替罪风险继续压住，不被快速解除
- 验证“同谋书生”空栏现实逼近王兆陪
- 验证主角只能制造延误或瑕疵，不能直接翻盘

## 批次风险点

- 盖印前流程写成行政说明
- 王兆陪靠一句话拖住公文
- 瘦书办过早识破全部假线
- 忘记邢管事仍被扣、鲁伙计仍不能公开

---

## 第161章：副册落名
- **功能**：压
- **目标字数**：2000~2500字
- **本章即时卖点**：邢管事的名字不只写在草稿上，还被抄入递签副册
- **本章兑现点**：新告文进入盖印前流程
- **最晚冲突落点**：前20%
- **核心事件**：王兆陪发现新告文已经从草稿转入递签副册，邢管事的名字被正式登记为“涉文中人”，盖印前流程开始
- **冲突类型**：文书递签 / 替罪风险升级
- **人物发展**：主角意识到不能再只盯草稿，必须阻断流程
- **信息释放**：新告文离正印只差核名、请印两步
- **钩子**：副册旁边还夹着一张空白核名单，上头预留“同谋书生”
- **本章不可删场景**：副册落名和流程推进必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角知道邢管事将被写成替罪中间人，自己可能进入同谋栏
  - 必做事件：
    1. 新告文进入递签副册
    2. 邢管事名被正式抄入副册
    3. 发现“同谋书生”核名单
  - 禁止越界：
    1. 不救出邢管事
    2. 不公开真残墨
    3. 不阻止整个流程
  - 结束状态：新告文进入盖印前流程
  - 下章承接：曹书吏尝试补王兆陪名
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 副册查见，负责压迫
    2. 递签流程判断，负责信息
    3. 同谋核名单，负责钩子
  - 字数支撑理由：流程推进有现实压力和下一步风险
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约850~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。需用现场动作避免行政说明。

## 第162章：同谋书生
- **功能**：压 / 争
- **目标字数**：2000~2500字
- **本章即时卖点**：曹书吏不写“王兆陪”，先写“王某”，逼他自己对号入座
- **本章兑现点**：同谋空栏开始现实逼近主角
- **最晚冲突落点**：前20%
- **核心事件**：曹书吏试图在核名单上补入“王某”，以模糊写法逼近王兆陪，同时保留改写空间
- **冲突类型**：模糊落名 / 名分逼近
- **人物发展**：主角意识到对方不必马上写全名，也能制造实际压力
- **信息释放**：对方准备先用“王某”通过核名，再在请印前补全
- **钩子**：核名单需要递送簿值手补一笔“曾收王某签押”
- **本章不可删场景**：王某落名和签押补笔必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：新告文进入盖印前流程，核名单预留同谋书生
  - 必做事件：
    1. 曹书吏尝试写“王某”
    2. 主角判断模糊落名的危险
    3. 签押补笔指向提前签收值手
  - 禁止越界：
    1. 不让主角当场撕毁核名单
    2. 不让曹书吏直接承认
    3. 不公开真残墨
  - 结束状态：主角必须借值手调动痕迹制造延误
  - 下章承接：利用提前签收值手
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 王某落名，负责压迫
    2. 模糊落名判断，负责人物
    3. 签押补笔，负责下一章入口
  - 字数支撑理由：名分逼近和签押补笔具备现实冲突
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约800~950字
    3. 场景3：约500~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。主角不能靠嘴，要靠流程瑕疵。

## 第163章：值手不在
- **功能**：争 / 得
- **目标字数**：2000~2500字
- **本章即时卖点**：王兆陪不拦告文，只让补签的人“不在原位”
- **本章兑现点**：主角制造盖印前小延误
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪利用提前签收值手被调走的痕迹，让核名单缺少必要补签，迫使新告文无法立即递入请印
- **冲突类型**：流程瑕疵 / 延误
- **人物发展**：主角开始用对手调动造成的漏洞反制
- **信息释放**：曹书吏调走值手本意是隔断红泥线，却意外造成核名补签缺口
- **钩子**：瘦书办察觉红泥拓痕假线可能有问题
- **本章不可删场景**：值手不在和请印延误必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：核名单需要递送簿值手补“曾收王某签押”
  - 必做事件：
    1. 主角确认值手已被调走
    2. 核名单因缺补签无法立即请印
    3. 瘦书办察觉红泥拓痕假线有疑点
  - 禁止越界：
    1. 不抓值手
    2. 不解决新告文
    3. 不公开真残墨
  - 结束状态：新告文被延误，但对手开始怀疑假线
  - 下章承接：瘦书办反查红泥拓痕
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 查值手去向，负责动作
    2. 补签缺口，负责兑现
    3. 瘦书办起疑，负责钩子
  - 字数支撑理由：不是口头拖延，而是流程缺口
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约850~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。需写出值手调动的因果。

## 第164章：假泥太新
- **功能**：压 / 争
- **目标字数**：2000~2500字
- **本章即时卖点**：瘦书办没有找到真残墨，却看出红泥拓痕太新
- **本章兑现点**：对手部分识破假线，但仍不知道真残墨位置
- **最晚冲突落点**：前25%
- **核心事件**：瘦书办复查红泥拓痕，发现其中一处泥色过新，判断王兆陪放出的不是全部证据，但仍未知道真残墨在外部保管
- **冲突类型**：假线被疑 / 信息差收窄
- **人物发展**：主角意识到假线不能再拖太久
- **信息释放**：瘦书办比曹书吏更敏锐，但仍缺真证方向
- **钩子**：瘦书办建议越过正印流程，先把新告文加急送上一级“请示”
- **本章不可删场景**：假泥太新和加急请示建议必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：新告文被补签缺口延误，对手开始怀疑假线
  - 必做事件：
    1. 瘦书办复查红泥拓痕
    2. 判断假泥太新
    3. 提出越过正印先加急请示
  - 禁止越界：
    1. 不让瘦书办找到真残墨
    2. 不让曹书吏完全醒悟
    3. 不解决新告文
  - 结束状态：假线开始失效，新告文转向上级请示
  - 下章承接：未盖正印但加急上送
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 复查拓痕，负责反压
    2. 假泥判断，负责信息
    3. 加急请示，负责钩子
  - 字数支撑理由：对手反应升级，主角压力变强
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约800~950字
    3. 场景3：约500~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。对手识破半步带来新压力。

    3. 不救出邢管事
