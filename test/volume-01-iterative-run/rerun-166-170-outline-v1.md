# 新模具复跑细纲 v1：第166~170章

> 用途：承接第165章“上级回批前必须找到被调走的值手”，验证追值手是否能从文书流程转成现场动作，并验证 `03-chapters.md` 新增的“流程章现场动作和人物后果”约束。

## 批次目标

- 验证值手从名单名字变成现场人物
- 验证主角只能取得半证，不能让值手完整交代
- 验证上级回批压力继续逼近
- 验证流程章每章都绑定现场动作或人物后果

## 批次风险点

- 值手成为一次性证人
- 上级回批写成抽象压力
- 签押样本直接翻盘
- 邢管事仍被扣、王某书生仍未洗清这两条被遗忘

---

## 第166章：外院库房
- **功能**：争
- **目标字数**：2000~2500字
- **本章即时卖点**：王兆陪追到外院库房，值手正被安排搬封存旧箱
- **本章兑现点**：值手从名单名字变成现场人物
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪按调名册找到外院库房，发现递送簿值手被安排搬运封存旧箱，周围有人盯着，不能直接问话
- **冲突类型**：现场追人 / 被盯值手
- **人物发展**：主角从流程判断转为现场取证
- **信息释放**：值手不是被正常调走，而是被放在能消耗体力、隔绝外人的活计里
- **钩子**：值手搬箱时手腕露出一道新压痕，像被迫按过印
- **本章不可删场景**：外院库房追人和搬封存旧箱必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：疑案急请已上送，主角必须在回批前找到值手
  - 必做事件：
    1. 追到外院库房
    2. 看见值手搬封存旧箱且被盯
    3. 发现手腕新压痕
  - 禁止越界：
    1. 不让值手完整交代
    2. 不公开真残墨
    3. 不阻止疑案急请
  - 结束状态：值手成为现场人物，但无法直接问话
  - 下章承接：从手伤和墨痕查被迫补笔
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 外院库房寻人，负责现场动作
    2. 值手搬封箱，负责人物后果
    3. 手腕压痕，负责钩子
  - 字数支撑理由：追人、盯梢、封箱和压痕能展开为现场对抗
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约800~950字
    3. 场景3：约500~650字
  - 摘要风险判断：低
  - 流程章落地检查：有现场动作，值手被安排搬箱是人物后果
  - 最低可撑体量判断：能。库房现场和被盯状态可支撑展开。

## 第167章：手上有墨
- **功能**：争 / 压
- **目标字数**：2000~2500字
- **本章即时卖点**：值手不开口，手上却有被迫补笔留下的墨痕
- **本章兑现点**：取得半证但不能直接作证
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪借搬箱递水接近值手，发现他右手虎口有新墨和压伤，值手只低声说“不是我写的”，便被盯梢人打断
- **冲突类型**：半证获取 / 盯梢打断
- **人物发展**：主角接受半证状态，不急于逼供
- **信息释放**：值手可能被迫按手或摹笔，但补签文字未必是他亲写
- **钩子**：盯梢人把值手调去封箱后门，明显要转移
- **本章不可删场景**：递水接近和盯梢打断必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角看到值手手腕压痕，但无法直接问话
  - 必做事件：
    1. 借递水接近值手
    2. 发现新墨和压伤
    3. 值手只说“不是我写的”后被打断
  - 禁止越界：
    1. 不让值手完整交代
    2. 不让主角当场救走值手
    3. 不公开真残墨
  - 结束状态：主角取得半证，值手即将被转移
  - 下章承接：上级催问先到，府内要求补齐王某书生
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 递水接近，负责动作
    2. 墨痕压伤，负责信息
    3. 盯梢打断，负责压力
  - 字数支撑理由：半证取得和被打断有现场张力
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约750~950字
    3. 场景3：约500~650字
  - 摘要风险判断：低
  - 流程章落地检查：补笔流程落到手伤、墨痕和盯梢打断
  - 最低可撑体量判断：能。现场接近和打断足够展开。

## 第168章：催问先到
- **功能**：压
- **目标字数**：2000~2500字
- **本章即时卖点**：王兆陪还没拿稳半证，上级催问已经先压到府里
- **本章兑现点**：上级回批压力具体落地
- **最晚冲突落点**：前20%
- **核心事件**：疑案急请的催问先到府里，要求三日内补齐“王某书生”身份，府内因此逼曹书吏补名，也逼王兆陪自证
- **冲突类型**：上级压力 / 身份补名
- **人物发展**：主角从追证转为同时抗压
- **信息释放**：上级不需要定罪，只要府里补齐疑名，压力就能落到王兆陪身上
- **钩子**：催问附条里写着“王某若不认，应取原签押样本对照”
- **本章不可删场景**：催问到府和补齐身份压力必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角取得值手半证，值手即将被转移
  - 必做事件：
    1. 上级催问到府
    2. 要求补齐王某书生身份
    3. 提出取原签押样本对照
  - 禁止越界：
    1. 不让主角用半证直接翻盘
    2. 不公开真残墨
    3. 不让值手公开作证
  - 结束状态：王某书生身份压力逼近，签押样本成为下一步入口
  - 下章承接：用值手半证拖住补名
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 催问到府，负责压力现场
    2. 补名逼迫，负责人物后果
    3. 签押样本对照，负责钩子
  - 字数支撑理由：上级压力落到府内人物行动，不是抽象说明
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约850~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：中
  - 流程章落地检查：催问流程绑定府内补名和主角自证压力
  - 最低可撑体量判断：能。催问、补名、签押对照能形成连续压力。

## 第169章：半证拖名
- **功能**：争 / 得
- **目标字数**：2000~2500字
- **本章即时卖点**：王兆陪不证明自己无罪，只证明“现在不能补名”
- **本章兑现点**：利用半证拖住补名
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪不能让值手公开作证，只用“值手手伤、补签人不在场、原签押待核”三点，迫使府内暂缓补全“王某书生”
- **冲突类型**：半证周旋 / 暂缓补名
- **人物发展**：主角学会不求翻案，只争取时间
- **信息释放**：半证不能洗清王兆陪，但能证明当前补名流程有瑕疵
- **钩子**：曹书吏决定当夜转走值手
- **本章不可删场景**：暂缓补名和不能公开值手必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：上级催问要求补齐王某书生身份
  - 必做事件：
    1. 主角提出补名流程有瑕疵
    2. 不公开值手，只用半证拖住补名
    3. 曹书吏决定转走值手
  - 禁止越界：
    1. 不让王兆陪彻底洗清
    2. 不让值手公开作证
    3. 不公开真残墨
  - 结束状态：补名暂缓，但值手转移风险升级
  - 下章承接：值手转移前留下签押样本
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 府内补名会场，负责对抗
    2. 半证拖名，负责兑现
    3. 曹书吏转走值手，负责钩子
  - 字数支撑理由：主角不是讲道理，而是在现场用瑕疵争时间
  - 场景级字数预算：
    1. 场景1：约700~850字
    2. 场景2：约850~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 流程章落地检查：补名流程绑定府内会场、值手保护和转移后果
  - 最低可撑体量判断：能。现场周旋和转移风险支撑章节。

## 第170章：改过的签押
- **功能**：得 / 压
- **目标字数**：2000~2500字
- **本章即时卖点**：值手被转走前没说真话，只留下一张被改过的签押样本
- **本章兑现点**：取得签押真假验证入口
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪赶到值手被转移的后门，只来得及接到一张被揉皱的签押样本，上面“王某”二字有改笔痕迹，但值手被带走
- **冲突类型**：转移现场 / 残证取得
- **人物发展**：主角接受只拿到不完整证据，继续保留追线
- **信息释放**：签押样本被改过，但不能直接证明是谁改
- **钩子**：签押样本背面有一枚极淡的上级催问封泥痕，说明样本可能被提前准备好
- **本章不可删场景**：后门转移和值手留下样本必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：补名暂缓，但曹书吏要转走值手
  - 必做事件：
    1. 主角赶到转移后门
    2. 值手留下改过的签押样本
    3. 值手被带走
  - 禁止越界：
    1. 不救下值手
    2. 不直接证明王兆陪清白
    3. 不公开真残墨
  - 结束状态：主角拿到签押真假验证入口
  - 下章承接：验证签押样本真假
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 后门追转移，负责动作
    2. 签押样本交接，负责兑现
    3. 封泥痕发现，负责钩子
  - 字数支撑理由：追人、交证、失人和新痕迹构成双峰
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约850~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 流程章落地检查：签押验证入口通过后门转移和样本交接落地
  - 最低可撑体量判断：能。转移现场和残证取得能稳定支撑。
