# 新模具复跑细纲 v1：第226~230章

> 用途：验证“临时封条来源”是否进入领用簿、封条样式、空白数量、交接人和实物痕迹的现场验证，而不是写成“尺寸吻合，所以来自急件房 / 府学”。

## 批次目标

- 验证封条来源必须落到领用簿、样式、存数、领用人和交接人
- 至少模拟一次临时封条卷紧后是否能塞入架底并造成半格错位
- 尺寸吻合只能证明某类物件可能进入封袋架，不能证明来源
- 府学来文不能过早坐实参与
- 王某疑名和府学查籍压力继续存在

## 批次风险点

- 尺寸吻合直接定来源
- 府学过早坐实为对手
- 急件房领用簿只停留在口头
- 王某疑名被写丢

---

## 第226章：领用空押

- **功能**：查 / 得
- **目标字数**：2000~2500字
- **本章即时卖点**：王兆陪查急件房临时封条领用簿，发现领用数量与存数差一条，领用人栏却空押
- **本章兑现点**：封条来源进入领用簿和数量现场
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪到急件房核对临时封条领用簿、柜内存数和废封条回收数，发现账面应余十二条，柜中只有十一条，缺口所在行领用人栏空押
- **冲突类型**：账簿核对 / 证据边界
- **人物发展**：主角从尺寸比对转向领用链验证
- **信息释放**：急件房确有一条临时封条缺口，但不能证明它进入封袋架
- **钩子**：空押行旁边的交接人只写了一个“门”字
- **本章不可删场景**：领用簿、存数核对、领用人空押必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：临时封条尺寸吻合半格，但来源未定
  - 必做事件：
    1. 查急件房临时封条领用簿
    2. 核对柜内存数与废封条回收数
    3. 发现领用人空押和交接人“门”字
  - 禁止越界：
    1. 不证明缺失封条就是架底垫物
    2. 不证明府学参与
    3. 不撤王某疑名
  - 结束状态：急件房存在空押缺口
  - 下章承接：比对府学来文封条样式
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 领用簿核对，负责账面
    2. 存数与废封条核对，负责实物数量
    3. 空押和“门”字，负责钩子
  - 字数支撑理由：账簿核对、急件房书办阻拦和数量差异能支撑展开
  - 场景级字数预算：
    1. 场景1：约700~850字
    2. 场景2：约750~900字
    3. 场景3：约550~700字
  - 摘要风险判断：低
  - 流程章落地检查：封条来源落到领用簿和存数
  - 最低可撑体量判断：能。

## 第227章：纸纹不同

- **功能**：查 / 争
- **目标字数**：2000~2500字
- **本章即时卖点**：比对府学来文封条，尺寸接近但纸纹不同，不能证明来自府学
- **本章兑现点**：府学来源被纳入验证，但不坐实
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪调看府学催文封条样式，发现宽度与急件房临时封条接近，但纸纹更细、胶口不同；府学来文可解释尺寸，却不能解释架底木屑和水盆黑灰
- **冲突类型**：样式比对 / 来源边界
- **人物发展**：主角阻止曹书吏把府学来文直接写成来源
- **信息释放**：府学封条只是相似来源，不是确定来源
- **钩子**：府学封条交接记录的时辰正好在午后查籍文书之前
- **本章不可删场景**：府学封条尺寸、纸纹、胶口比对必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：急件房少一条临时封条，交接人栏有“门”字
  - 必做事件：
    1. 调看府学来文封条
    2. 比对尺寸、纸纹、胶口
    3. 标出查籍文书前的交接时辰
  - 禁止越界：
    1. 不证明封条来自府学
    2. 不证明府学参与陷害
    3. 不证明急件房无关
  - 结束状态：府学来文成为相似但未定来源
  - 下章承接：模拟临时封条卷紧塞入架底
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 府学封条调看，负责对象
    2. 纸纹胶口比对，负责差异
    3. 交接时辰，负责压力
  - 字数支撑理由：实物比对、曹书吏借府学施压和时辰线能支撑
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约850~1000字
    3. 场景3：约500~650字
  - 摘要风险判断：低
  - 流程章落地检查：府学线不靠相似定案，落到纸纹和胶口
  - 最低可撑体量判断：能。

## 第228章：卷封入架

- **功能**：查 / 压
- **目标字数**：2000~2500字
- **本章即时卖点**：模拟临时封条卷紧塞入架底，能造成半格错位，并沾上黑灰和木屑
- **本章兑现点**：封条从“来源线”进入“能否成为架底垫物”的动作验证
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪将急件房同款临时封条卷紧，塞入封袋架底，再抽出。它能顶出半格，抽出时发出木响，表面可沾上架底木屑和水盆黑灰，但纸面折痕与昨夜旧压痕还差一线
- **冲突类型**：动作模拟 / 半证推进
- **人物发展**：主角确认临时封条可作为垫物，但不定来源
- **信息释放**：临时封条比薄木片更能解释半格、木响和痕迹复合
- **钩子**：模拟封条的折痕方向和旧压痕相反，说明昨夜那条可能从另一侧塞入
- **本章不可删场景**：卷封条、塞架底、沾黑灰木屑、折痕差异必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：急件房和府学来文都只是可能来源
  - 必做事件：
    1. 卷紧临时封条
    2. 塞入封袋架底并抽出
    3. 检查黑灰、木屑和折痕方向
  - 禁止越界：
    1. 不证明缺失封条就是它
    2. 不证明府学参与
    3. 不解除王某疑名
  - 结束状态：临时封条可解释架底垫物，但来源未定
  - 下章承接：曹书吏借来源未定压查籍
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 卷封条，负责物件形态
    2. 塞入抽出，负责动作复现
    3. 折痕差异，负责边界
  - 字数支撑理由：实物操作、痕迹复合和半证边界能支撑
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约850~1000字
    3. 场景3：约550~700字
  - 摘要风险判断：低
  - 流程章落地检查：封条来源线进入动作验证
  - 最低可撑体量判断：能。

## 第229章：来源未定

- **功能**：压 / 争
- **目标字数**：2000~2500字
- **本章即时卖点**：曹书吏压查籍，王兆陪用“封条类物件可能存在，但来源未定”阻止补名
- **本章兑现点**：封条半证进入程序对抗
- **最晚冲突落点**：前20%
- **核心事件**：曹书吏抓住折痕差异与来源未定，要求按午后时限移交府学查籍；王兆陪承认来源未定，却指出来源未定不能写成王兆陪涉案，也不能抹掉急件房空押缺口
- **冲突类型**：会场对抗 / 程序威胁
- **人物发展**：主角继续用证据边界拖住补名和查籍
- **信息释放**：对手想把来源未定转为主角无法自证
- **钩子**：赵平认出“门”字可能不是门房，而是门禁册的简称
- **本章不可删场景**：查籍施压、来源未定边界、门禁册钩子必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：临时封条可解释架底垫物，但来源未定
  - 必做事件：
    1. 曹书吏压府学查籍
    2. 主角用来源未定反压补名
    3. 赵平补出“门”可能指门禁册
  - 禁止越界：
    1. 不解除查籍压力
    2. 不证明门禁册已改
    3. 不证明具体领用人
  - 结束状态：封条来源线转向急件房空押和门禁册
  - 下章承接：收束封条来源，下一批查空押领用人和急件房门禁
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 查籍施压，负责外部压力
    2. 来源未定反压，负责冲突
    3. “门”字解释转向门禁册，负责钩子
  - 字数支撑理由：程序压力、证据边界和新账册方向能支撑
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约850~1000字
    3. 场景3：约500~650字
  - 摘要风险判断：低
  - 流程章落地检查：封条半证产生程序后果
  - 最低可撑体量判断：能。

## 第230章：门禁缺口

- **功能**：得 / 压
- **目标字数**：2000~2500字
- **本章即时卖点**：封条来源线只能证明急件房有空押缺口，下一步转向空押领用人和急件房门禁
- **本章兑现点**：封条来源阶段收束，进入下一轮门禁验证
- **最晚冲突落点**：前25%
- **核心事件**：王兆陪复盘领用簿空押、府学封条差异、卷封条模拟和来源未定边界，确认只能证明急件房临时封条有缺口；他查急件房门禁简册，发现同一时段有一条入房记录只写“急件取封”，没有人名
- **冲突类型**：证据边界 / 新方向转移
- **人物发展**：主角接受封条来源半证，继续向门禁和领用人推进
- **信息释放**：空押缺口与门禁无名记录可能相连
- **钩子**：无名门禁记录时辰，正好卡在府学查籍文书送入前一刻
- **本章不可删场景**：封条来源收束、急件房门禁简册、无名记录时辰必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：封条类物件可解释架底垫物，但来源未定
  - 必做事件：
    1. 收束封条来源证据边界
    2. 查急件房门禁简册
    3. 发现无名“急件取封”记录
  - 禁止越界：
    1. 不证明取封人身份
    2. 不证明府学参与
    3. 不撤王某疑名
  - 结束状态：下一批转向空押领用人和急件房门禁
  - 下章承接：查急件房门禁记录对应值门人
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 封条来源收束，负责边界
    2. 门禁简册查询，负责新线索
    3. 无名时辰卡点，负责钩子
  - 字数支撑理由：半证复盘、门禁查询和查籍时限能支撑
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约750~900字
    3. 场景3：约650~800字
  - 摘要风险判断：中
  - 流程章落地检查：封条来源不作终局推理，转向门禁现场
  - 最低可撑体量判断：能。
