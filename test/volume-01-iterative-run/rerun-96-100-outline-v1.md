# 新模具复跑细纲 v1：第96~100章

> 用途：承接 `91~95` 已验证过的卷尾高风险段，继续测试“现实追索 / 多线承接 / 高信息密度收束”在连续 5 章里的稳定性。

## 批次目标

- 验证工作流在连续承接章里，能否避免重新退回摘要稿
- 验证“证据线 / 人证线 / 路线认知线”三条线并行时，章纲是否仍可写
- 验证体量略高时，是否依然可以通过轻压缩处理，而不必回退到重写骨架

## 批次风险点

- 连续追索章容易写成“到处打听”的流水账
- 多线并行时容易只剩解释，缺乏场景动作
- 高信息密度章容易为了交代而牺牲章末钩子

---

## 第96章：血鞋印不等人
- **功能**：争
- **目标字数**：2000~2500字
- **本章即时卖点**：鲁伙计刚失踪半日，留下的血鞋印就已经把主角逼到必须连夜追出去
- **本章兑现点**：把“鲁伙计出险”从消息变成可见现场，并让主角第一次沿着现实危险线主动出手
- **最晚冲突落点**：前25%
- **核心事件**：主角、邢管事沿鞋印和铜扣线追索；途中确认这不是普通失足，而是有人专门收拾尾巴
- **冲突类型**：现实追索 / 人证出险
- **人物发展**：王兆陪第一次把“写证”的抽象认知压回现实救人
- **信息释放**：带血鞋印和半枚铜扣不是偶然遗落，而是追人、拖人时留下的硬痕
- **钩子**：河边草窠里翻出一截被撕掉的账纸角，和旧帖边角的纸性对得上
- **本章不可删场景**：夜追血鞋印那一段必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角刚得知鲁伙计失踪，证据线与人证线同步出险
  - 必做事件：
    1. 把失踪现场写实，不能只靠转述
    2. 让主角和邢管事通过追索确认“这不是普通跑失”
    3. 在追索尽头抛出能接证据线的新残片
  - 禁止越界：
    1. 不直接救回鲁伙计
    2. 不直接给出幕后主使
  - 结束状态：主角确认自己已经被拖进更深一层的收尾局
  - 下章承接：围绕新残片确认鲁伙计为何值得被人灭口
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 失踪现场与第一轮判断，负责把危机落地
    2. 沿线追索与途中试探，负责把危险写实
    3. 河边残片入手，负责把证据线重新接上
  - 字数支撑理由：本章是完整追索事件链，自带现场、行动、判断和钩子，不依赖解释撑字
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约800~1000字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。每段都包含动作与新信息，不会天然滑成摘要。

## 第97章：残纸背后有人改账
- **功能**：争 / 压
- **目标字数**：2000~2500字
- **本章即时卖点**：河边那截残纸不光证明鲁伙计见过东西，还证明有人提前动过账
- **本章兑现点**：把“残纸角”和旧帖边角、仓务线索拼成同一条链
- **最晚冲突落点**：前35%
- **核心事件**：主角用两轮场景化验证残纸来源，并确认这条线已经不止牵到鲁伙计个人
- **冲突类型**：证据拼接 / 线索升级
- **人物发展**：主角开始明白，自己追的不是单独一个人，而是一整套改账和收口的路数
- **信息释放**：残纸不是普通账页，而是经过改写、拆散、再销痕的一部分
- **钩子**：老先生递来一句话：“有人怕的不是你知道，是你写得成。”
- **本章不可删场景**：两轮不同身份的人对残纸给出不同分量的确认
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角手里拿着和旧帖线索能对上的残纸
  - 必做事件：
    1. 通过两轮验证确认残纸来源
    2. 让主角意识到“人证出险”是为了给改账线灭口
    3. 落下“写得成”这句更高层提醒
  - 禁止越界：
    1. 不直接拿到整张账页
    2. 不直接让常先生明面出手
  - 结束状态：主角确认自己已经摸到真正会致命的一层
  - 下章承接：围绕“要不要先护人还是先护证”进入取舍
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 屋内比对与初判，负责立证据价值
    2. 外出求证与身份差反应，负责把残纸砸实
    3. 老先生递话，负责把证据线与路线线重新扣上
  - 字数支撑理由：本章是验证过程章，不是结论章，天然依赖多轮对照与主角判断推进
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约750~950字
    3. 场景3：约450~650字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。两轮验证都具备独立增量。

## 第98章：先护人，还是先护证
- **功能**：压 / 争
- **目标字数**：2000~2500字
- **本章即时卖点**：线索越清，主角越发现自己只能二选一：先救人，还是先护证
- **本章兑现点**：把“路线认知”第一次逼成现实选择，而不是停留在想法层面
- **最晚冲突落点**：前30%
- **核心事件**：主角与邢管事、老先生立场分化，围绕先追鲁伙计还是先藏证据展开争执和布置
- **冲突类型**：策略取舍 / 多线承接
- **人物发展**：主角第一次需要自己定次序，而不是跟着别人给的线走
- **信息释放**：常先生不出手并不等于不在看，反而像在等他自己排出轻重
- **钩子**：夜半传回消息，鲁伙计可能还活着，但只会在天亮前转一回窝
- **本章不可删场景**：三人对“先护人还是先护证”的分歧必须落在具体布置上
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角已确认人证与证据都不能再拖
  - 必做事件：
    1. 把不同立场摆出来，不能只让主角一个人想
    2. 让主角做出先后取舍，并安排对应动作
    3. 章末抛出时限更短的救人入口
  - 禁止越界：
    1. 不在本章直接把鲁伙计救回
    2. 不在本章直接丢掉证据线
  - 结束状态：主角带着明确选择进入更直接的行动章
  - 下章承接：围绕转窝消息做一轮更近距离的碰撞
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 回来整线与分歧抛出，负责立选择压力
    2. 三方争执与布置，负责把策略冲突写实
    3. 夜半急报入门，负责压出行动钩子
  - 字数支撑理由：本章不是空谈取舍，而是“分歧 - 定序 - 接报”三段连动
  - 场景级字数预算：
    1. 场景1：约600~750字
    2. 场景2：约850~1050字
    3. 场景3：约450~650字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。分歧本身带动作决策，不会只剩说理。

## 第99章：转窝之前
- **功能**：争
- **目标字数**：2000~2500字
- **本章即时卖点**：天亮前只剩一回转窝机会，主角必须在最短时间内摸到人还在不在
- **本章兑现点**：把“救人窗口”真正写成短时限行动，而不是普通打探章
- **最晚冲突落点**：前25%
- **核心事件**：主角借夜色、旧识和半枚铜扣线逼近转窝地点，第一次和收尾的人擦身而过
- **冲突类型**：限时探线 / 近距离碰撞
- **人物发展**：主角开始学会在不知道全局的情况下先做局部判断
- **信息释放**：收尾的人训练有素，不像临时起意，更像长期替人清线
- **钩子**：破屋里没找到鲁伙计，只找到一截被扯断的衣襟和一句没写完的字
- **本章不可删场景**：和收尾人近距离擦身那一段必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角知道天亮前只有一次机会
  - 必做事件：
    1. 通过有限线索逼近转窝点
    2. 写出一次近距离但未正面开战的碰撞
    3. 在落空中留下一条更硬的新入口
  - 禁止越界：
    1. 不直接揭开收尾人身份
    2. 不直接找到鲁伙计本人
  - 结束状态：主角确认自己追到的已不是边角，而是对方真正的收口动作
  - 下章承接：围绕断衣襟和未写完的字进入阶段收束
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 逼近转窝点，负责拉时间压迫
    2. 近距离擦身与判断，负责制造现实危险
    3. 破屋落空与新线索入手，负责把失败转成新入口
  - 字数支撑理由：本章是短时限行动章，自带动作、误判风险和现场反馈
  - 场景级字数预算：
    1. 场景1：约650~800字
    2. 场景2：约750~950字
    3. 场景3：约500~700字
  - 摘要风险判断：低
  - 最低可撑体量判断：能。每一段都依赖场景推进。

## 第100章：写成，才压不回去
- **功能**：破 / 得
- **目标字数**：2000~2500字
- **本章即时卖点**：人还没救回，主角却先被逼着做出第一次真正意义上的“写证”
- **本章兑现点**：把前几章追来的残纸、断字、铜扣和人证风险，第一次组织成可往上递的雏形
- **最晚冲突落点**：前35%
- **核心事件**：主角回收现有碎证，决定先写出一份压不回去的简证底稿；常先生仍不现身，只借一句话把这一章收住
- **冲突类型**：阶段收束 / 路线第一次落笔
- **人物发展**：主角第一次从“知道该怎么写”迈到“真的开始写”
- **信息释放**：真正可怕的不是证据稀少，而是证据在没写成之前都能被压回去
- **钩子**：底稿刚成，门外就有人递来一句：“写得出来，也得送得出去。”
- **本章不可删场景**：主角把碎证第一次组织成底稿的过程必须落地
- **视角**：王兆陪
- **节奏**：双峰
- **写作交接卡**：
  - 开场状态：主角手里已有多段碎证，但人证仍未脱险
  - 必做事件：
    1. 回收前四章的关键碎证并重新排布
    2. 把“写证”第一次落成实际动作，而不是理念复述
    3. 章末留下一句更高层的送递难题
  - 禁止越界：
    1. 不直接完成整案翻盘
    2. 不直接让常先生现身收人
  - 结束状态：主角已经从追证推进到第一次正式落笔
  - 下章承接：围绕“写出来以后怎么送出去”进入新阶段
- **章节可写性证明**：
  - 场景数：3
  - 场景拆分：
    1. 回收碎证与确定写法，负责收束前段信息
    2. 正式起稿与删改，负责把理念变成动作
    3. 门外递话，负责把阶段性收束转成新难题
  - 字数支撑理由：本章虽然信息密度高，但不是总结章，而是“收证 - 起稿 - 留难”三段推进
  - 场景级字数预算：
    1. 场景1：约650~850字
    2. 场景2：约850~1050字
    3. 场景3：约400~600字
  - 摘要风险判断：中
  - 最低可撑体量判断：能。核心场景是实际落笔动作，不靠回顾性解释硬拖。

## 本批次备注
- **批次范围**：第96~100章
- **批次目标**：验证连续承接章、多线并行章和阶段收束章，是否仍能保持可写性与成稿体量
- **批次风险点**：第98章和第100章最容易退成“纯讨论 / 纯总结”，正文试写时应优先观察是否场景化不足
- **对下一批的约束**：如果这 5 章里再次出现“体量够但动作虚”的问题，优先回查 `03-chapters.md` 对多线承接章的场景拆分要求
