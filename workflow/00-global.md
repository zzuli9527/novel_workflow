# Phase 0：修仙搞笑项目全局设定

> 目的：把用户创意整理为可持续写作的项目设定、修炼规则、喜剧圣经和顶层故事单元。

## Step 0：输入盘点

### 最低输入

- 主角或核心视角角色
- 基本修仙创意
- 开局处境
- 希望的搞笑方向
- 目标篇幅范围

### 可选输入

- 结局方向
- 世界观
- 修炼体系
- 金手指或特殊能力
- 核心配角
- 长期关系
- 主要势力
- 长期对手
- 平台和语言风格
- 禁止剧情
- 禁止笑料

### 输入盘点 Prompt

```text
你是一位修仙搞笑长篇网文策划师。

请先盘点用户输入，不要直接开始扩写剧情。

用户输入：
{{用户输入}}

输出：

1. 用户已经明确提供的信息
2. 仍然缺失的信息
3. 可以安全补全的项目
4. 必须让用户确认后才能确定的重大项目
5. 每项信息来源：用户明确 / 模型建议 / 待确认

不得覆盖用户已经确认的设定。
不得预设具体境界名称、宗门、法宝或结局。
```

通过条件：

- 用户输入与模型补全明确区分。
- 重大设定没有被静默决定。
- 开始下一步所需信息齐全或已列出候选。

## Phase 0-A：故事承诺与终点方向

本阶段回答：这本书长期让读者期待什么，主角最终要走到哪里。

### 必须确定

- 主角当前最缺少什么
- 主角长期想得到什么
- 修仙成长的主要吸引点
- 搞笑风格的主要吸引点
- 主角不能轻易放弃的核心关系或原则
- 长期问题的最终收束方向
- 结局应保留的情感或关系结果

### Prompt

```text
你是一位长篇修仙搞笑网文结构师。

输入：
{{输入盘点结果}}

任务：

如果用户尚未给出终点方向，提出 3 个可持续的候选方向。
如果用户已经给出终点，提供 2 个结构变化方案和 1 个保守校验方案。

每个方案必须包含：

- 一句话故事承诺
- 主角长期目标
- 修仙成长终点
- 核心关系终点
- 搞笑风格如何持续到中后期
- 最终需要解决的核心问题
- 不应提前解决的内容
- 主要风险

不要写具体章节。
不要用空泛的“成为最强”代替人物目标。
```

## Phase 0-B：修炼体系与成长边界

### 必须确定

1. 境界或成长层级
2. 每个层级的能力边界
3. 突破条件
4. 突破代价
5. 资源种类和稀缺程度
6. 功法、法术和法宝的基本规则
7. 伤势与恢复规则
8. 越级行为的允许条件
9. 全书成长上限
10. 已确认例外

### 原则

- 上限确定后不能无说明增加更高层级。
- 能力提升必须带来新问题或新责任，不只增加数字。
- 低级资源不能在后期无限失效，必要时说明替代关系。
- 突破不能仅靠偶然笑料完成。
- 金手指必须有边界、成本或使用条件。
- 同一能力的效果必须保持一致。

### Prompt

```text
你是一位修仙体系设计与连续性编辑。

输入：

项目资料：
{{项目资料}}

用户已有修炼设定：
{{已有设定，没有则留空}}

任务：

在继承用户设定的前提下，输出：

1. 境界或成长层级表
2. 每层能力边界
3. 大致进度范围
4. 突破条件与代价
5. 资源体系
6. 功法、法术、法宝规则
7. 伤势与恢复规则
8. 越级条件
9. 金手指边界
10. 禁止清单
11. 来源标注

搞笑事故可以影响突破过程，但不能绕过既定条件。
```

## Phase 0-C：喜剧圣经

### 必须确定

- 整体喜剧密度：轻喜 / 中等 / 高密度
- 主角的喜剧角色定位
- 核心配角各自的反差来源
- 主要喜剧机制
- 可以回调的长期笑点
- 不允许使用的笑料
- 严肃场景的喜剧上限
- 语言风格和现代词汇边界

### 角色喜剧定位

每个核心角色至少记录：

- 认真目标
- 最在意的体面或原则
- 容易失控的条件
- 与其他角色的反差
- 说话方式
- 不能被破坏的人物底线

### Prompt

```text
你是一位情境喜剧设计师，同时理解修仙长篇的连续性。

输入：

项目资料：
{{项目资料}}

核心角色：
{{角色资料}}

用户希望的搞笑方向：
{{搞笑方向}}

任务：

建立本项目的喜剧圣经：

1. 喜剧密度
2. 主角的喜剧定位
3. 核心配角的反差和语言习惯
4. 允许使用的主要喜剧机制
5. 可长期回调的内容
6. 禁止笑料
7. 严肃场景的喜剧上限
8. 搞笑疲劳预警
9. 前 20 章的机制轮换建议

必须保证：

- 角色不是为了段子临时变笨
- 搞笑结果会影响关系、信息或行动
- 不依赖网络梗堆砌
- 不取消修仙世界的真实危险
```

## Phase 0-D：顶层故事单元

顶层规划不直接铺满全书细纲，而是建立连续的阶段单元。

每个故事单元建议覆盖约 10～20 章。

### 每个故事单元必须包含

- 单元标题
- 章节范围建议
- 进入状态
- 阶段目标
- 主要阻碍
- 修仙成长变化
- 资源变化
- 关系变化
- 信息差或误会变化
- 喜剧机制计划
- 至少一次真实受挫或代价
- 阶段兑现
- 退出状态
- 不得提前解决的长期问题
- 与下一单元的因果接口

### Prompt

```text
你是一位修仙搞笑长篇网文总纲师。

输入：

故事承诺与终点：
{{Phase 0-A}}

修炼体系：
{{Phase 0-B}}

喜剧圣经：
{{Phase 0-C}}

目标篇幅：
{{目标篇幅}}

任务：

规划顶层故事单元。

要求：

1. 每个单元约 10～20 章，具体数量由事件密度决定。
2. 每个单元都要有明确进入状态和退出状态。
3. 修仙成长与喜剧机制必须共同推进。
4. 不连续安排只有准备、没有结果的单元。
5. 每个单元至少有一次代价、一次可见兑现和一个后续问题。
6. 不把搞笑线独立成与主线无关的小品。
7. 标注用户提供、模型补全和待确认内容。

输出顶层结构，不生成逐章正文。
```

## Phase 0 输出清单

```text
项目定位
故事承诺
终点方向
修炼体系
资源与代价规则
喜剧圣经
核心角色喜剧定位
顶层故事单元表
项目禁止清单
来源记录
初始结构化状态：角色境界、资源余额、按角色区分的知识状态
```

初始结构化状态必须同步写入 `config/initial-state.json`。角色、资源和事实使用后续章节不变的稳定 ID；资源必须包含持有者、数量和单位。

- [ ] 修炼体系边界已经确认
- [ ] 金手指或特殊能力边界已经确认
- [ ] 喜剧密度和禁用笑料已经确认
- [ ] 当前故事单元有明确目标和退出状态
- [ ] 当前故事单元没有提前解决长期问题
- [ ] 所有模型补全的重要设定已经标记
