# Phase 1：故事单元展开

> 目的：把一个约 10～20 章的故事单元展开成连续、可映射为章节的故事点。

## 输入

```text
项目定位
修炼体系与成长边界
喜剧圣经
最近状态快照（续写时）
最近批次账本（续写时）
当前故事单元
模块库
```

## Prompt

```text
你是一位修仙搞笑长篇网文结构师。

你的任务是展开当前故事单元，不写正文，不重新设计整本书。

### 项目资料
{{项目定位}}

### 修炼体系
{{修炼体系}}

### 喜剧圣经
{{喜剧圣经}}

### 当前状态
{{最近状态快照；首个单元则使用开局状态}}

### 最近账本
{{没有则写“无”}}

### 当前故事单元
{{故事单元}}

### 模块库
{{modules.md}}

请将当前故事单元展开为连续故事点。
```

## 展开规则

### 1. 每个故事点必须有认真目标

角色必须正在完成具体事情，例如：

- 获取或保护资源
- 解决修炼问题
- 完成任务
- 处理关系或势力条件
- 调查世界规则
- 应对身份、伤势或能力限制

不能只写“角色之间发生有趣互动”。

### 2. 每个故事点选择模块

记录：

- 主要修仙推进模块
- 主要喜剧机制
- 可选回调

同一主要喜剧机制连续使用两次后，下一故事点应更换机制，或明显改变对象、后果和场景。

### 3. 因果必须连续

相邻故事点要能回答：

```text
上一个故事点的哪个结果，迫使角色进入下一个故事点？
```

禁止只靠“与此同时”“几天后”“另一边”连接互不相关的事件。

### 4. 修仙主线必须变化

每个故事点至少推动一项：

- 能力认知
- 修炼进度
- 资源
- 伤势或限制
- 身份与势力位置
- 关系
- 世界规则
- 长期问题的解决进度

只有笑点、没有状态变化的内容不能单独成为故事点。

### 5. 搞笑必须参与因果

搞笑触发点必须造成至少一个结果：

- 改变角色判断
- 暴露或隐藏信息
- 改变关系
- 迫使角色换方案
- 引发新的成本
- 提供解决问题的意外入口

删掉搞笑内容后，因果链完全不变，说明搞笑仍是外挂段子，需要重做。

### 6. 代价与后果

一个故事单元至少包含一次真实代价或受挫。

代价可以是：

- 资源消耗
- 伤势
- 时间损失
- 身份暴露
- 关系恶化
- 新义务
- 错误判断
- 暂时失去机会

搞笑不能自动清除代价。

### 7. 叶节点判定

满足以下条件时可标记为叶节点：

- 有明确进入状态和退出状态。
- 有一个主要问题。
- 有可执行行动，而非抽象目标。
- 有修仙主线变化。
- 有主要喜剧机制或明确的低喜剧说明。
- 预计可映射为 1～3 章。
- 再拆会变成单纯动作步骤，而不是独立叙事事件。

不满足时继续展开。

### 8. 批次节奏

整个故事单元必须做到：

- 前段尽快建立问题和行动。
- 中段至少一次条件变化或真实受挫。
- 后段兑现主要承诺。
- 结尾形成可见成长结果。
- 保留一个由本单元结果产生的后续问题。
- 不连续两个故事点只做准备。
- 不连续三个故事点只搞笑、不推进修仙主线。

## 硬失败

- 故事点之间没有因果。
- 只有搞笑互动，没有主线状态变化。
- 突破或能力变化违反修炼体系。
- 重要资源凭空出现。
- 角色为了笑点突然降智。
- 预计章节数主要依赖对话注水。
- 故事单元没有可见兑现。
