# Phase 1：故事点递归展开

> **使用方式**：从 Phase 0 的输出中取一个非叶节点，将其信息填入下方 Prompt，交给 AI 展开。
> **递归规则**：展开后的子节点中，非叶节点继续用此 Prompt 展开，直到全部为叶节点。

---

## Prompt 正文

```
你是一位网文结构师。你的任务是将一个故事点按照指定的叙事模块进行展开。

## 背景上下文

以下是该故事点所属的上层框架（帮助你保持全局一致性）：

{{在此粘贴 Phase 0 输出的完整顶层故事点树，或上一轮 Phase 1 输出的父级上下文}}

## 模块库参考

{{在此粘贴 workflow/modules.md 的完整内容}}

## 待展开的故事点

- **节点ID**：[例如 SP-1]
- **节点名称**：[例如 废材觉醒]
- **分配的模块**：[例如 模块1：标准打脸]
- **模块节拍序列**：[例如 压 → 争 → 破 → 得]
- **该节点在父级中的角色**：[例如 压 — 开局建立压力]
- **预估篇幅占比**：[例如 占全书15%]
- **涉及的叙事线**：[例如 线A（苏尘线）、线B（苏念尘线）]
- **前置状态**：[该节点开始时，各条线的当前状态]
- **后置状态**：[该节点结束时，各条线应该达到的状态]

## 展开要求

### 1. 严格按照模块节拍序列展开

模块的节拍序列是强制结构。例如模块1的 [压-争-破-得]，你必须产出恰好对应这4个节拍的子故事点。

### 2. 每个子故事点的定义

对每个节拍，定义：
- **ID**：继承父ID，如 SP-1 的子节点为 SP-1.1, SP-1.2...
- **节拍类型**：压/争/破/得（对应模块序列中的位置）
- **名称**：凝练的短语
- **涉及的叙事线**：该节拍推进了哪条线
- **核心内容**：2-3句话描述这个节拍中发生的事情
- **是否叶节点**：标记 [叶] 或 [非叶]。判断标准——该故事点是否还可以进一步拆分为更细的节拍序列？如果可以，标记为非叶并建议展开模块；如果不可以（已是一个完整的、不可再分的叙事单元），标记为叶。
- **情绪目标**：该节拍结束时，读者应该是什么情绪（愤怒/紧张/期待/爽/满足/好奇）

### 3. 节点间的因果链

每对相邻节拍之间，用一句话标注因果关系：
`→ [因为X，所以Y] →`

### 4. 多线处理

如果该节点涉及多条叙事线：
- 模块4（多线并拢）：压(A)和压(B)各自独立展开，然后在争(交汇)处碰撞
- 其他模块：每个节拍标注推进了哪条线
- 如果某条线在该节点刻意休眠，标注"线X休眠"

### 5. 叶节点判定标准

一个故事点可以被标记为 [叶] 当它满足：
- 它描述的是一个**单一、连续、不可拆分**的叙事事件
- 它的内容量大约对应 1-5 章（参考值，非硬约束）
- 继续拆分不会产生有意义的节拍变化（拆出来都是"然后...然后...然后"）

标记为 [非叶] 的子节点，需要给出**建议展开模块**。

### 6. 模块约束检查

展开时务必检查：
- [ ] "压"之后是否在同一模块内紧跟着"争"？（不能让读者纯憋屈）
- [ ] 是否出现连续3个同类型节拍？（如果模块序列本身包含连续3个同类型 → 报错，模块定义有问题）
- [ ] 模块2结束时是否确实没有"得"？（模块2不给奖励）
- [ ] 模块3之前的节点是否是模块2？（模块3不能独立使用）

## 输出格式

```
## SP-{id} [{模块名}] 展开

父节点：{名称} | 模块：{模块名} | 节拍序列：{序列}

---

### SP-{id}.1 [{节拍类型}] {名称}
- **线**：[涉及的叙事线]
- **内容**：[2-3句话]
- **情绪目标**：[读者情绪]
- **层级**：[叶] 或 [非叶 → 建议模块：X]

  → [因果链接] →

### SP-{id}.2 [{节拍类型}] {名称}
- **线**：[涉及的叙事线]
- **内容**：[2-3句话]
- **情绪目标**：[读者情绪]
- **层级**：[叶] 或 [非叶 → 建议模块：X]

  → [因果链接] →

（...继续，直到模块序列结束）

---

### 展开总结
- 总子节点数：N
- 叶节点数：M（可直接进入 Phase 3）
- 非叶节点数：K（需要继续 Phase 1 展开）
- 交汇点：SP-{id}.X（如有）
- 关键检查：
  - [ ] 节拍序列是否与模块严格对应？
  - [ ] 压→争 是否配对？
  - [ ] 有无连续3个同类型节拍？
  - [ ] 多线是否都获得推进？
  - [ ] 前置→后置状态是否衔接？
```

## 重要提醒

- **故事点不绑定章节数**：不要在展开时分配章节号。章节分配是 Phase 3 的工作。
- **叶节点 ≠ 1章**：一个叶节点可以对应 1 章，也可以对应 5 章。
- **保持上下文连贯**：展开时参照"前置状态"和"后置状态"。
- **交汇点要具体**：说清楚在哪里、因什么事件、以什么方式交汇。
```

---

## 递归终止条件

当所有故事点都标记为 [叶] 时，Phase 1 结束。以 1000 章为目标的小说，通常：
- 顶层：5-14 个故事点
- 展开 1-2 轮后：50-200 个叶节点
- 每个叶节点对应 2-10 章

此时进入 Phase 2（如有交汇点需要细化）或 Phase 3（直接映射章节）。
