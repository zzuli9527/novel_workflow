# Phase 2：多线因果交汇

> 目的：让修炼、资源、关系、势力和信息差通过同一事件发生因果碰撞，而不是轮流汇报。

本阶段按需执行。单主线可以跳过。

## 输入

```text
当前故事单元
需要交汇的故事点
各线当前状态
修炼体系
喜剧圣经
```

## 常见叙事线

- 修炼或能力线
- 资源线
- 任务或探索线
- 势力线
- 关系线
- 身份或秘密线
- 误会与知识差线

搞笑不是独立叙事线。它必须作用于上述某条线或交汇事件。

## 交汇模式

### M1：共同资源

两条线需要同一个资源、地点、人物或机会，被迫相撞。

### M2：行动连锁

A 线的行动直接改变 B 线条件，B 线的反应又反过来影响 A 线。

### M3：信息错位

不同角色基于不同信息行动，最终在同一现场产生冲突、误会或意外合作。

### M4：共同期限

两条线必须在同一个时间节点前完成各自目标，因此出现资源和选择冲突。

### M5：身份与关系碰撞

角色在不同关系中的身份同时生效，必须作出取舍。

### M6：一线引爆另一线

一条线的结果揭开、破坏或提前触发另一条线。

## Prompt

```text
你是一位修仙搞笑长篇的多线结构编辑。

输入：

当前故事单元：
{{故事单元}}

待交汇线：
{{各线信息}}

当前状态：
{{状态}}

修炼体系：
{{修炼体系}}

喜剧圣经：
{{喜剧圣经}}

任务：

选择最合适的交汇模式，把多条线改写为一条连续因果链。

要求：

1. 每条线进入交汇前有独立目标。
2. 交汇由行动、资源、信息、期限或身份触发。
3. 搞笑机制参与交汇，但不取消严肃后果。
4. 最多同时处理两条主线；三条以上先分组交汇。
5. 交汇后必须产生新的共同状态。
6. 不依赖频繁无目的切视角制造复杂感。
```

## 微节拍规则

每个微节拍记录：

- 当前视角
- 角色目标
- 触发动作
- 获得的信息
- 丢失或改变的条件
- 喜剧机制
- 对另一条线的影响
- 新状态

视角切换必须发生在信息、行动或场面控制权变化处。

## 硬失败

- 多条线只是依次出现，没有互相改变。
- 交汇完全依赖巧合，且此前没有铺垫。
- 搞笑角色只负责打断主线。
- 某条线进入后没有目标、没有结果。
- 交汇结果违反已确认状态。
