# Phase 2：多线交汇处理

> **使用方式**：当 Phase 1 的输出中存在**交汇点**（多条叙事线在同一故事点汇合），用此 Prompt 进行细化。
> **触发条件**：一个故事点涉及 ≥2 条叙事线，且需要设计它们如何交叉推进。

## 核心概念：交叉模式

多条线交汇时，有几种基本交叉模式：

### 模式1：乒乓（交替推进）
```
线A → 线B → 线A → 线B → 汇合
```
适用于：两条线势均力敌，互相影响。读者在两条线之间来回切换。

### 模式2：螺旋（逐步靠近）
```
线A ──┐
       ├── 线A&B 接触 ──┐
线B ──┘                 ├── 汇合
线C ────────────────────┘
```
适用于：多条线从独立推进到逐渐接触再到完全汇合。模块4（多线并拢）的标准展开方式。

### 模式3：引爆（一条线打破另一条）
```
线A ────────────┬── 线A被冲击
                │
线B ── 引爆点 ──┘
```
适用于：一条线的事件突然闯入另一条线，打破原有节奏。常用于"转"节拍。

### 模式4：同步（平行剪辑）
```
线A：事件A1 → 事件A2 → 事件A3
线B：事件B1 → 事件B2 → 事件B3
（最后汇合）
```
适用于：多条线在同步推进但尚未接触时，通过平行剪辑制造"命运交汇"的预感。

## 输出要求

### 1. 选择交叉模式
为该交汇点选择 1-2 种交叉模式，说明选择理由。

### 2. 细化节拍序列
将该交汇点拆分为更细的节拍序列（微节拍），每个微节拍标注：
- **序号**：微节拍在该节点内的顺序
- **推进的线**：线A/线B/线C/多线
- **微节拍类型**：压/争/破/得（在微观层面的功能）
- **内容**：1-2句话描述这个微节拍中发生的事情
- **视角**：该微节拍从谁的视角展开（主角/对手/上帝视角）
- **情绪**：读者此刻应该感受到什么

### 3. 交叉节奏图
用时间线展示各线的交叉节奏：
```
微节拍:  #1    #2    #3    #4    #5    #6
线A:     ██          ██          ██
线B:           ██          ██          ████ (汇合)
线C:     ████████████████████████
情绪:    期待  好奇  紧张  紧张  震撼  释放
```

## 使用时机

并非所有交汇点都需要 Phase 2。判断标准：

| 情况 | 是否需要 Phase 2 |
|------|------------------|
| 交汇点已经是 [叶] 节点 | **需要** — 交汇是最复杂的叙事操作，叶节点交汇必须细化 |
| 交汇点是 [非叶] 节点 | **先展开再判断** — Phase 1 展开后，看子节点是否还有交汇，如有则需要 |
| 单线节点 | **不需要** — 直接走 Phase 1 → Phase 3 |
| 交汇极其简单（如只是擦肩而过） | **可跳过** — 在 Phase 3 中顺带处理 |
