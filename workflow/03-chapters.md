# Phase 3：故事点 → 章节映射 + 章级细纲

> **使用方式**：取一个或多个连续的叶节点故事点，用此 Prompt 生成章节级细纲。
> **建议粒度**：每次输入 1-3 个叶节点（约 5-20 章的量），保证 AI 有足够上下文且输出不溢出。

---

## Prompt 正文

```

## 章节映射规则

### 1. 故事点 → 章节数量的判断

一个叶节点应该映射为多少章，取决于：
- **内容浓度**：该故事点包含几个关键转折/冲突/揭示？每个关键节拍通常需要 1-2 章来展开。
- **节奏需求**：如果该故事点处于模块2（螺旋施压），章节可以偏多（拖长压抑感）。如果处于模块3（集中收割），章节应该紧凑（快速释放）。
- **多线因素**：如果涉及多线交叉，每切换一次视角至少需要半章到一章。
- **读者体验**：确保每章都有"获得感"（信息增量/爽点/悬念），不能出现"水章"。

为了让映射结果更稳定，建议先做一轮**章节预算**，再生成具体章纲。

#### 建议预算规则

先按叶节点节拍类型给一个基础章数范围：

| 节拍类型 | 基础章数 |
|----------|----------|
| 压 | 1-3章 |
| 争 | 1-4章 |
| 破 | 1-2章 |
| 得 | 1-2章 |

然后按以下因素修正：

- **模块修正**
  - 模块2（螺旋施压）：总章数可在基础上 +1 到 +2
  - 模块3（集中收割）：总章数可在基础上 -1，但至少保留 1 章
  - 模块5（无缝钩子）：通常控制在 1 章，最多 2 章
- **多线修正**
  - 涉及 2 条线：总章数 +1
  - 涉及 3 条线及以上：总章数 +2，但要警惕注水
- **信息修正**
  - 如果该叶节点承担重大揭示、重大翻盘、重大关系变化，可 +1
  - 如果只是承接动作，不承担新信息释放，不要人为拉长

#### 预算使用原则

- 先给每个叶节点一个**预算章数**
- 再在预算范围内生成具体章节
- 如果生成后的章纲明显超出预算，优先压缩过渡章，而不是继续加章
- 如果一个叶节点 1 章写不完，必须说明是因为包含多个关键动作或多线交汇，而不是“写着写着变长了”

### 2. 章节结构

每章细纲必须包含：
- **章标题**：（可先写临时标题）
- **本章功能**：压/争/破/得（在微观层面，服务于父模块的宏观节拍）
- **核心事件**：这一章发生的核心事件（1-2句话）
- **冲突类型**：人与人的对抗 / 人与环境的对抗 / 人与自己的对抗 / 信息揭示
- **人物发展**：主角或关键人物在这一章中有什么变化（能力/认知/关系/地位）
- **信息释放**：这一章向读者揭示了什么新信息（世界观/伏笔/人物背景）
- **章节钩子**：章末的悬念或期待点（让读者翻下一章）
- **视角**：从谁的视角展开

### 3. 节奏控制

- **单一高潮章**：整章为一个高潮事件服务（常用于"破"和"得"）
- **双峰章**：章内有两个情绪峰值（一个小高潮 + 铺垫下一章）
- **过渡章**：承上启下，信息密度较低但必要（尽量少用，每 5-8 章最多 1 章过渡）
- **多线章**：本章内切换 ≥2 个视角（标注切换点）

### 4. 写作交接卡

章节细纲不是终点。为了让下一步正文写作稳定执行，每章细纲后都要附一张交接卡，作为正文写作的固定输入。

交接卡至少包含：

- **本章开场状态**：这一章开始时，主角/关键角色分别处于什么状态
- **本章必须完成的事件**：这章最少要完成什么，少一项都算没写到位
- **本章禁止越界内容**：不能提前揭示什么，不能提前突破什么，不能提前和解什么
- **本章结束状态**：这章收尾时，人物关系、局势、信息状态应变成什么
- **下章承接钩子**：下一章要接什么，不允许写成封口章

## 全局约束

在生成章节细纲时，请时刻检查：

- [ ] **每章必须有钩子**：读者需要理由翻下一章。钩子可以是悬念、未解决的冲突、新信息的一角。
- [ ] **信息释放节奏**：不是每章都能爆大料。信息释放应有层次——暗示 → 线索 → 部分揭示 → 完全揭示。
- [ ] **情绪不能平**：连续 3 章以上没有情绪起伏视为节奏问题。
- [ ] **视角清晰**：如果多线，每次视角切换要明确标注。
- [ ] **前置/后置衔接**：第一张章开始时状态 = 前置状态，最后一张章结束时状态 = 后置状态。
- [ ] **模块一致性**：章节的微观节拍应服务于父模块的宏观节拍。如果父模块是 2（螺旋施压），这些章就不应该有"得"。如果父模块是 3（集中收割），就不应该有"压"。
- [ ] **预算不漂移**：生成章数应尽量落在预算范围内；如超出，说明原因。
```

## 写作后记录

正文写完后，不要直接进入下一章。先做一轮回填，至少记录：

- **人物状态变化**：谁升级了、受伤了、立场变了、目标变了
- **关系变化**：谁与谁更亲近、更对立、达成合作、彻底决裂
- **战力/资源变化**：境界、法宝、金手指、情报、势力资源有什么变化
- **新伏笔**：本章新埋了什么
- **已回收伏笔**：本章解决了哪些旧钩子
- **下章前置状态**：下一章开始前，读者和角色分别知道什么、不知道什么

如果回填后发现正文和细纲明显不一致：

1. 优先微调正文
2. 如正文更合理，再回改本章细纲
3. 连续 3 章以上都偏了，再回查对应叶节点

不要跳过这一步。长篇写作最容易失控的地方，不是不会展开，而是写完后没有把变化记下来。

## 批量处理建议

如果你的叶节点较多（如 50+ 个），建议分批处理：

1. **按大段分批**：一次处理一个顶层故事点下的所有叶节点（约 10-30 章）
2. **保持连续性**：每批处理时，把上一批的最后一章状态作为输入
3. **章号接力**：手动维护一个章号计数器，每批结束后更新
