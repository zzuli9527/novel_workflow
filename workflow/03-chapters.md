# Phase 3：故事点 → 章节映射 + 章级细纲

> **使用方式**：取一个或多个连续的叶节点故事点，用此 Prompt 生成章节级细纲。
> **建议粒度**：每次输入 1-3 个叶节点（约 5-20 章的量），保证 AI 有足够上下文且输出不溢出。

> **整卷实跑补充**：如果目标不是样章，而是第一卷 80~120 章级别的连续实跑，不要一次性把整卷细纲全部写死。应拆成多个连续批次（建议每批 5~10 章），每批生成后立即进入正文测试、状态回填、批次复查，再决定是否调整后续细纲。

```
## 章节映射规则

### 1. 故事点 → 章节数量的判断

一个叶节点应该映射为多少章，取决于：
- **内容浓度**：该故事点包含几个关键转折/冲突/揭示？每个关键节拍通常需要 1-2 章来展开。
- **节奏需求**：如果该故事点处于模块2（螺旋施压），可以增加压力层次，但不能靠拖慢事件推进制造压抑感；如果处于模块3（集中收割），章节应该紧凑（快速释放）。
- **多线因素**：如果涉及多线交叉，每切换一次视角至少需要半章到一章。
- **读者体验**：确保每章都有"获得感"（信息增量/爽点/悬念），不能出现"水章"。

### 1.5 字数反推规则（重要）

默认每章目标字数为**2000~2500字**。因此章节映射不能只看节拍，还要反推单章是否有足够内容支撑。

判断原则：

- 一个章节至少应承载下列内容中的 2-3 项：
  - 事件推进
  - 冲突升级
  - 信息释放
  - 人物关系变化
  - 章末钩子
- 一个章节还必须至少存在 1 个**读者可感知的实质变化**，例如：
  - 主角拿到新线索
  - 主角做成一个关键动作
  - 对手态度被逼出来
  - 关系或局势明显变化
  - 危机被具体升级
- 如果某个叶节点只够支撑一个很小的动作，就不要硬拆成一章
- 如果某章主要内容只是“发现一点异常 → 想一会儿 → 再确认一点”，原则上不应单独撑成一章
- 如果某一章预计只能靠反复描写、空转心理、重复解释来凑字数，说明章节规划有问题，应优先：
  1. 合并章节
  2. 合并叶节点
  3. 回查 Phase 1 是否拆得过碎

为了让映射结果更稳定，建议先做一轮**章节预算**，再生成具体章纲。

#### 建议预算规则

先按叶节点节拍类型给一个基础章数范围：

| 节拍类型 | 基础章数 |
|----------|----------|
| 压 | 1-3章 |
| 争 | 1-4章 |
| 破 | 1-2章 |
| 得 | 1-2章 |

然后按以下因素修正：

- **模块修正**
  - 模块2（螺旋施压）：总章数可在基础上 +1 到 +2
  - 模块3（集中收割）：总章数可在基础上 -1，但至少保留 1 章
  - 模块5（无缝钩子）：通常控制在 1 章，最多 2 章
- **多线修正**
  - 涉及 2 条线：总章数 +1
  - 涉及 3 条线及以上：总章数 +2，但要警惕注水
- **信息修正**
  - 如果该叶节点承担重大揭示、重大翻盘、重大关系变化，可 +1
  - 如果只是承接动作，不承担新信息释放，不要人为拉长

#### 预算使用原则

- 先给每个叶节点一个**预算章数**
- 再在预算范围内生成具体章节
- 如果生成后的章纲明显超出预算，优先压缩过渡章，而不是继续加章
- 如果一个叶节点 1 章写不完，必须说明是因为包含多个关键动作或多线交汇，而不是“写着写着变长了”
- 如果某章天然不够支撑 2000~2500 字，应优先合并，而不是硬保章数
- 连续两章如果都只有施压、没有兑现点，视为节奏风险，应优先合并、压缩，或提前释放小回报

### 2. 章节结构

每章细纲必须包含：
- **章标题**：（可先写临时标题）
- **本章功能**：压/争/破/得（在微观层面，服务于父模块的宏观节拍）
- **目标字数**：默认 2000~2500 字
- **本章即时卖点**：前 300~800 字最先抓住读者的东西（危机 / 利益 / 羞辱 / 异常 / 赌局 / 爽点）
- **本章兑现点**：本章必须落地的一个实质回报，不能整章只铺不兑
- **最晚冲突落点**：主冲突最晚应在本章前多少比例落地；首章必须更早
- **核心事件**：这一章发生的核心事件（1-2句话）
- **冲突类型**：人与人的对抗 / 人与环境的对抗 / 人与自己的对抗 / 信息揭示
- **人物发展**：主角或关键人物在这一章中有什么变化（能力/认知/关系/地位）
- **信息释放**：这一章向读者揭示了什么新信息（世界观/伏笔/人物背景）
- **章节钩子**：章末的悬念或期待点（让读者翻下一章）
- **本章不可删场景**：本章最该被读者记住的动作场景 / 对峙场景 / 爽点场景
- **视角**：从谁的视角展开

### 3. 节奏控制

- **单一高潮章**：整章为一个高潮事件服务（常用于"破"和"得"）
- **双峰章**：章内有两个情绪峰值（一个小高潮 + 铺垫下一章）
- **过渡章**：承上启下，信息密度较低但必要（尽量少用，每 5-8 章最多 1 章过渡）
- **多线章**：本章内切换 ≥2 个视角（标注切换点）

### 3.5 首章特殊规则

第1章必须单独处理，不能按普通章节等量看待。

首章应尽快完成：

1. 主角亮相
2. 主角现状建立
3. 核心异常 / 冲突出现
4. 让读者愿意继续看的吸引点落下

首章硬指标：

- 前 300~500 字内，至少完成“主角是谁 / 主角现状 / 眼前麻烦”中的 2 项
- 前 800~1200 字内，必须落下第一次明确冲突、异常、利益点或危险
- 首章不能只停在“主角意识到问题”，必须给出一个可见动作、判断或对抗

禁止：

- 先铺设定再讲人
- 用日常流水账慢慢带入
- 把前几百字浪费在背景说明
- 只有氛围没有事件

### 4. 写作交接卡

章节细纲不是终点。为了让下一步正文写作稳定执行，每章细纲后都要附一张交接卡，作为正文写作的固定输入。

交接卡至少包含：

- **本章开场状态**：这一章开始时，主角/关键角色分别处于什么状态
- **本章必须完成的事件**：这章最少要完成什么，少一项都算没写到位
- **本章禁止越界内容**：不能提前揭示什么，不能提前突破什么，不能提前和解什么
- **本章结束状态**：这章收尾时，人物关系、局势、信息状态应变成什么
- **下章承接钩子**：下一章要接什么，不允许写成封口章

### 4.5 章节可写性证明（新增硬约束）

为了避免章纲表面合格、正文实际只能写成摘要，每章细纲后必须额外输出一段**章节可写性证明**。

至少包含：

- **场景数**：本章准备由几个有效场景组成（通常不少于 2 个）
- **场景拆分**：每个场景分别承担什么功能
- **字数支撑理由**：为什么这些场景合起来足够自然支撑 2000~2500 字
- **摘要风险判断**：这一章是否存在“只够写成概述 / 复盘 / 过桥段”的风险

#### 场景拆分最低要求

- 一个可写成正文章节的章纲，通常至少应包含下列中的 2 项以上：
  - 现场冲突
  - 场景对话
  - 动作处理
  - 信息交换
  - 关系变化
  - 章末转折
- 如果本章只能列出：
  - 1 个小动作
  - 1 次简短判断
  - 1 个章末钩子
  
  则默认**不够单独成章**

#### 场景级字数预算（新增硬约束）

仅仅证明“本章有 2~3 个场景”还不够，还必须证明这些场景各自**能展开**。

因此每章都必须补充**场景级字数预算**，至少说明：

- 每个场景预计承载的主要内容
- 每个场景为什么不是一句话就能交代完
- 每个场景预计贡献的大致体量

建议格式：

- 场景1：约 600~900 字
- 场景2：约 700~1000 字
- 场景3：约 500~800 字

#### 场景预算判定原则

若一个场景本质上只是：

- 一次极短信息告知
- 一句口信
- 一轮很短的判断
- 一次轻量态度变化

则该场景默认**不能单独承担高体量预算**。

如果整章的场景预算加总后，只有靠“心理补白 / 总结补白 / 解释补白”才能勉强靠近 2000 字，则该章纲仍然视为**不合格**。

#### 薄场景判废（新增硬约束）

以下情况，即使表面上已经不是“摘要章”，也不得视为合格章纲：

1. 全章只有 1 个主场景，且该场景本质只是信息确认、短会面、态度试探或一句口信传递
2. 全章只有 2 个短场景，但场景之间没有新的动作推进、关系变化、利益交换或外部打断
3. 所谓“还能继续展开”的空间，主要依赖重复解释、心理空转、气氛拉长或换说法复述同一判断
4. 去掉心理补白和环境描写后，剩余有效动作链明显偏短

一旦命中上述任一条，本章应直接标记为**薄场景章**，默认动作不是进入正文，而是：

1. 补新场景
2. 与前后章合并
3. 重拆本段目标，让本章承担更实质的兑现

#### 高风险章节类型补充

下面几类章节最容易看似有内容、实际天然偏短，必须额外加严检查：

1. **信息确认章**
   - 例如：确认一张纸、一段口供、一个名字
   - 若没有至少 2 轮场景化验证或冲突，不足以单独成章

2. **理念交锋章**
   - 例如：两人坐着谈“怎么赢”“替谁写”
   - 若只有一轮高密度对话、没有额外动作插入、关系变化或危机打断，天然容易偏短

3. **人物亮相章**
   - 例如：某个关键人物正式出场
   - 若只有“介绍此人 + 说几句话 + 抛命题”，天然容易退成摘要稿

4. **文书 / 流程 / 制度章**
   - 例如：递签、核名、请印、盖印、上报、审查、登记、调令、榜文流转
   - 章纲不能只写“进入流程 / 递签 / 请印 / 上报”
   - 必须至少绑定一个可写成现场的动作或人物后果：
     1. 谁在现场翻册、补签、递文、改名、盖印、拦文、送文
     2. 谁的位置、名声、资源或行动空间因此被压缩
     3. 谁被调走、被扣住、被迫补笔、被迫背责或被迫改口
   - 如果本章只有制度名词推进，没有明确人物动作和后果，默认判为**流程说明章**，不能直接进入正文
   - 同一批次内不得连续重复使用“册子被调走 / 记录被取走 / 账本被封存 / 夹板被拿走”作为主要阻断。若必须使用文书或记录阻断，必须同步给出：
     1. 具体执行人或受影响人
     2. 主角损失的时间、机会或证据
     3. 新追查方向或新的现场动作

5. **卷尾收束章**
   - 例如：盘点收获、总结阶段成果、下一卷入口
   - 若没有明确场景承载，默认会滑向总结腔

#### 这些高风险章必须满足的附加条件

若本章属于上述高风险类型，至少满足其一：

1. 增加一个可独立展开的动作场景
2. 增加一轮有真实张力的关系对话
3. 增加一次外部打断、危机插入或利益交换
4. 如仍不足，则与前后章节合并处理

#### 必须合并的情况

出现以下任一情况，优先回到章纲阶段合并，不允许硬保章数：

1. 本章主要内容是“得知一点信息 + 想一会儿 + 留个钩子”
2. 本章主要内容是“复盘上一章 + 交代去下一章”
3. 本章只靠一场极短会面或一句口信维持存在
4. 本章如果展开到 2000~2500 字，只能依赖重复解释、心理空转、总结性复述
5. 本章虽可勉强写成“正文样子”，但按场景预算保守推演，稳定体量仍明显落在 800~1400 字区间

#### 最低可撑体量证明（新增硬约束）

每章在输出“场景级字数预算”后，还必须额外补一句：

- **最低可撑体量判断**：本章按当前场景设计，保守展开时是否仍能稳定支撑接近目标体量？

判断规则：

1. 若答案是“不能”，本章不得进入 `04-draft.md`
2. 若答案是“勉强能”，必须明确指出依赖的是哪些具体场景动作，而不是“文风展开”或“细节描写”
3. 若答案建立在“可以多写心理 / 可以多写气氛 / 可以多写解释”上，视为**判断无效**
4. 若保守推演后稳定体量仍明显偏低，应优先视为章纲预算失败，而不是正文执行问题

#### 单章装载过满提示（新增软约束）

为了避免从“太薄”直接走向“太满”，每章在证明“能写满”之后，还要顺手检查是否**装得过多**。

优先复查以下组合是否同时出现在同一章里：

1. 关键人物正式亮相
2. 新规则或新路线被完整抛出
3. 关系试探或站位试探出现明显推进
4. 一个可独立成立的外部危机被植入
5. 本章末尾还额外承担下一章开局应做的铺垫

若同一章里同时出现其中 **3 项及以上**，默认进入**装载过满复查**：

1. 先判断这些内容里，哪些属于本章必须兑现
2. 再判断哪些其实更适合挪到下一章开头
3. 不要为了“看起来更扎实”把两个章级兑现点强塞进一章

这不是硬性失败条件，但如果章纲作者自己已经明显感觉“这一章什么都重要、什么都想放”，通常说明应该先拆层，而不是直接进入正文。

#### 卷级实跑补充

如果目标是第一卷连续实跑，每批章纲输出后，必须先做一轮**可写性排雷**：

- 标出本批哪些章最容易写成摘要稿
- 标出哪些章天然该合并
- 标出哪些章需要补场景才能进入正文

没有完成这一步，不应直接进入 `04-draft.md`

### 5. 整卷连续实跑规则

当目标是“第一卷完整测试”而不是“小样章验证”时，额外执行：

1. **先定卷级阶段，不先写死全部章节**
   - 先给出第一卷的 4~6 个阶段目标
   - 再按阶段拆成多个连续批次
   - 每批只细化当前 5~10 章

2. **每批细纲都要带批次目标**
   - 这一批 5~10 章总体要完成什么
   - 这一批结束后，主角状态应推进到哪里
   - 这一批最该暴露哪类风险（如：开局慢、卖点疲劳、多线发散、配角功能化）

3. **后续批次允许受前批测试结果反向修正**
   - 如果正文测试证明某条线过弱、某个配角过空、某类卖点疲劳过快
   - 后续批次细纲应允许微调
   - 但不应轻易推翻顶层阶段目标

4. **阶段收口必须可见**
   - 每 10~20 章至少形成一次阶段性收口
   - 每 20~30 章至少形成一次身份、资源、局势或地图层级的跃迁
   - 第一卷末必须形成明确“下一卷入口”

5. **批次级回归检查**
   - 生成当前批细纲时，必须回看上一批的正文问题
   - 如上一批暴露出“兑现弱 / 钩子弱 / 配角空 / 重复卖点”，当前批细纲必须显式规避

## 输出格式
```

```
## 全局约束

在生成章节细纲时，请时刻检查：

- [ ] **每章必须有钩子**：读者需要理由翻下一章。钩子可以是悬念、未解决的冲突、新信息的一角。
- [ ] **每章必须先有卖点**：不能先平铺说明，再慢慢进入本章重点。
- [ ] **每章必须有兑现点**：哪怕只是小兑现，也不能整章只铺垫。
- [ ] **信息释放节奏**：不是每章都能爆大料。信息释放应有层次——暗示 → 线索 → 部分揭示 → 完全揭示。
- [ ] **情绪不能平**：连续 3 章以上没有情绪起伏视为节奏问题。
- [ ] **连续施压有限制**：连续 2 章都只有“压”没有任何实质回报，视为追读风险。
- [ ] **视角清晰**：如果多线，每次视角切换要明确标注。
- [ ] **前置/后置衔接**：第一张章开始时状态 = 前置状态，最后一张章结束时状态 = 后置状态。
- [ ] **模块一致性**：章节的微观节拍应服务于父模块的宏观节拍。如果父模块是 2（螺旋施压），这些章就不应该有"得"。如果父模块是 3（集中收割），就不应该有"压"。
- [ ] **预算不漂移**：生成章数应尽量落在预算范围内；如超出，说明原因。
- [ ] **字数可支撑**：本章事件量是否足够自然支撑 2000~2500 字？
- [ ] **场景是否足够成章**：是否至少能拆成 2 个有效场景，而不是只有“动作 + 判断 + 钩子”？
- [ ] **场景预算是否真实**：每个场景是否真的能展开，而不是只能靠补解释凑字？
- [ ] **是否属于高风险短章类型**：信息确认章 / 理念交锋章 / 人物亮相章 / 文书流程章 / 卷尾收束章是否已加严处理？
- [ ] **流程章是否落地**：若涉及文书 / 流程 / 制度推进，是否绑定了现场动作和人物后果，而不是只写制度名词？
- [ ] **是否会天然写成摘要稿**：如果高概率只能写成总结/过桥段，应先合并或补场景。
- [ ] **首章是否抓人**：如果是第1章，主角/现状/冲突/吸引点是否足够快到位？
- [ ] **是否在用微动作撑章**：如果本章主要内容只是轻微发现、重复推理、轻微确认，应回收合并。
```

## 写作后记录

正文写完后，不要直接进入下一章。先做一轮回填，至少记录：

- **人物状态变化**：谁升级了、受伤了、立场变了、目标变了
- **关系变化**：谁与谁更亲近、更对立、达成合作、彻底决裂
- **战力/资源变化**：境界、法宝、金手指、情报、势力资源有什么变化
- **新伏笔**：本章新埋了什么
- **已回收伏笔**：本章解决了哪些旧钩子
- **下章前置状态**：下一章开始前，读者和角色分别知道什么、不知道什么

如果回填后发现正文和细纲明显不一致：

1. 优先微调正文
2. 如正文更合理，再回改本章细纲
3. 连续 3 章以上都偏了，再回查对应叶节点

不要跳过这一步。长篇写作最容易失控的地方，不是不会展开，而是写完后没有把变化记下来。

## 批量处理建议

如果你的叶节点较多（如 50+ 个），建议分批处理：

1. **按大段分批**：一次处理一个顶层故事点下的所有叶节点（约 10-30 章）
2. **保持连续性**：每批处理时，把上一批的最后一章状态作为输入
3. **章号接力**：手动维护一个章号计数器，每批结束后更新
4. **边写边修**：如果某批正文测试暴露明显问题，不要继续批量铺后续细纲；先修 `04-draft.md` / `05-update-state.md` / 当前批细纲，再做回归测试。
