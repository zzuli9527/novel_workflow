# Phase 3：故事点 → 章节映射 + 章级细纲

> **使用方式**：取一个或多个连续的叶节点故事点，用此 Prompt 生成章节级细纲。
> **建议粒度**：每次输入 1-3 个叶节点（约 5-20 章的量），保证 AI 有足够上下文且输出不溢出。

---

```
## 章节映射规则

### 1. 故事点 → 章节数量的判断

一个叶节点应该映射为多少章，取决于：
- **内容浓度**：该故事点包含几个关键转折/冲突/揭示？每个关键节拍通常需要 1-2 章来展开。
- **节奏需求**：如果该故事点处于模块2（螺旋施压），章节可以偏多（拖长压抑感）。如果处于模块3（集中收割），章节应该紧凑（快速释放）。
- **多线因素**：如果涉及多线交叉，每切换一次视角至少需要半章到一章。
- **读者体验**：确保每章都有"获得感"（信息增量/爽点/悬念），不能出现"水章"。

### 2. 章节结构

每章细纲必须包含：
- **章标题**：（可先写临时标题）
- **本章功能**：压/争/破/得（在微观层面，服务于父模块的宏观节拍）
- **核心事件**：这一章发生的核心事件（1-2句话）
- **冲突类型**：人与人的对抗 / 人与环境的对抗 / 人与自己的对抗 / 信息揭示
- **人物发展**：主角或关键人物在这一章中有什么变化（能力/认知/关系/地位）
- **信息释放**：这一章向读者揭示了什么新信息（世界观/伏笔/人物背景）
- **章节钩子**：章末的悬念或期待点（让读者翻下一章）
- **视角**：从谁的视角展开

### 3. 节奏控制

- **单一高潮章**：整章为一个高潮事件服务（常用于"破"和"得"）
- **双峰章**：章内有两个情绪峰值（一个小高潮 + 铺垫下一章）
- **过渡章**：承上启下，信息密度较低但必要（尽量少用，每 5-8 章最多 1 章过渡）
- **多线章**：本章内切换 ≥2 个视角（标注切换点）
```

```
## 全局约束

在生成章节细纲时，请时刻检查：

- [ ] **每章必须有钩子**：读者需要理由翻下一章。钩子可以是悬念、未解决的冲突、新信息的一角。
- [ ] **信息释放节奏**：不是每章都能爆大料。信息释放应有层次——暗示 → 线索 → 部分揭示 → 完全揭示。
- [ ] **情绪不能平**：连续 3 章以上没有情绪起伏视为节奏问题。
- [ ] **视角清晰**：如果多线，每次视角切换要明确标注。
- [ ] **前置/后置衔接**：第一张章开始时状态 = 前置状态，最后一张章结束时状态 = 后置状态。
- [ ] **模块一致性**：章节的微观节拍应服务于父模块的宏观节拍。如果父模块是 2（螺旋施压），这些章就不应该有"得"。如果父模块是 3（集中收割），就不应该有"压"。
```

## 批量处理建议

如果你的叶节点较多（如 50+ 个），建议分批处理：

1. **按大段分批**：一次处理一个顶层故事点下的所有叶节点（约 10-30 章）
2. **保持连续性**：每批处理时，把上一批的最后一章状态作为输入
3. **章号接力**：手动维护一个章号计数器，每批结束后更新
