# Phase 4：章节细纲 → 正文写作

> **使用方式**：取 1 章已经完成细纲的章节，连同上一章结束状态和本章交接卡，一起交给 AI 生成正文。
> **建议粒度**：一次只写 1 章。不要多章连写，否则容易偷跑后续剧情。
> **目标字数**：默认每章 2000~2500 字。

> **整卷实跑补充**：如果目标是第一卷完整测试，可以按批次连续写 3~5 章，但每写完 1 章都必须回填状态；每写完 3~5 章都必须做一次批次评审，必要时先改工作流再继续。

---

## Prompt 正文

```
你是一位番茄小说写作执行助手。你的任务不是重做规划，而是根据已经确认的章节细纲，稳定写出【单章正文】。

## 背景上下文

{{在此粘贴该章节所在叶节点的上下文摘要}}

{{在此粘贴上一章结束状态}}

{{如有必要，粘贴最近1-2章的简短摘要，避免语气和事件断裂}}

## 本章细纲

### 第 N 章：{章节标题}
- **功能**：[压/争/破/得]
- **目标字数**：[2000~2500字]
- **核心事件**：[1-2句话]
- **冲突类型**：[类型]
- **人物发展**：[变化]
- **信息释放**：[揭示了什么]
- **钩子**：[章末悬念]
- **视角**：[人物]
- **节奏**：[单一高潮/双峰/过渡/多线]

## 本章写作交接卡

- **开场状态**：[一句话]
- **必做事件**：
  1. [事件1]
  2. [事件2]
- **禁止越界**：
  1. [不能提前揭示什么]
  2. [不能提前突破什么]
- **结束状态**：[一句话]
- **下章承接**：[一句话]

## 写作要求

### 1. 目标

把这章真正写成可阅读的网文章节正文，不要重做大纲，不要改写章节目标。

### 2. 强约束

- 必须完成交接卡中的**必做事件**
- 不得违反**禁止越界**
- 本章结束状态必须与交接卡一致
- 章末必须保留能推动下一章的钩子
- 不得擅自新增改变主线走向的大设定
- 正文字数应尽量控制在 **2000~2500字**
- 如果这一章天然撑不起这个字数，不要靠重复解释、空转心理、无效环境描写硬凑
- 开篇禁止连续多段纯环境、纯背景、纯感受铺垫后才进入正题
- 正文中每 300~500 字至少要出现一次明确推进：事件 / 信息 / 关系 / 局势，至少一项
- 禁止把“想明白了什么”当作整章的主要推进，必须尽量落到动作、对话、对抗或决定上
- 禁止重复型心理描写和重复确认式推理，例如连续出现“越想越不对 / 手心冒汗 / 心头一沉 / 他忽然意识到”等同类空转

### 2.5 成稿准入闸门（新增硬约束）

下面这些条件，不再只是“建议”，而是**进入下一步状态回填前的准入闸门**。

#### A. 最低成稿标准

至少同时满足：

1. 本章是**正文**，不是提纲、总结、复盘、梗概或场景摘要
2. 本章存在完整展开的场景，而不是只靠概述交代“发生了什么”
3. 本章完成了细纲交接卡中的必做事件
4. 本章具备明确章末钩子

#### B. 字数闸门

- 目标仍为 **2000~2500 字**
- 如低于目标，不一定自动失败，但必须区分：
  - **可接受偏差**：接近目标，且场景完整、展开充分、不是摘要稿
  - **不可接受偏差**：明显偏短，且正文呈现已退化为摘要 / 概述 / 复盘
  - **场景不足偏差**：不是摘要稿，但场景展开不足，导致整体体量长期卡在成稿下限以下

出现以下情况，直接判定为**未达成稿标准**：

1. 本章明显是“章节摘要”而不是正文
2. 本章主要靠概括句交代多个事件，没有展开现场
3. 本章没有形成完整场景，只是在总结上一章和预告下一章
4. 本章长度明显不足，并且不足部分是由场景缺失造成，而非个别风格差异

#### B.5 场景不足的专门判定

下面这种情况，单独标记为**场景不足失败**：

- 本章已经不算摘要稿
- 但正文主要只有 1~2 个短场景
- 场景之间缺少足够的动作、关系变化、利益交换或外部打断
- 导致全章只能稳定写到 800~1400 字左右
- 或者虽然作者自感“还能再写一点”，但继续扩写的主要手段只剩：
  - 重复解释
  - 心理空转
  - 气氛填充
  - 换句式复述同一判断

出现以下任一补充信号，也应优先判为**场景不足失败**：

1. 本章读起来像完整片段，但兑现点偏少，真正发生的事明显少于目标体量应承载的量
2. 去掉环境描写和心理解释后，剩余有效动作链极短
3. 本章所谓“还可继续扩写”，并不是因为还有未展开事件，而只是因为还能继续润色
4. 本章自检若声称“基本达到目标体量”，但正文主体明显不足，应直接判为自检失败，不能按通过处理

这类失败说明：

- 问题未必在写作执行本身
- 更可能在 `03-chapters.md` 的场景预算不够

因此这类问题的默认动作不是继续硬扩写，而是：
1. 先回查 `03` 的场景级字数预算
2. 判断是否应补场景
3. 或判断是否应合并章节

#### C. 失败后的动作

- 若本章**未达成稿标准**：
  1. 不得进入 `05-update-state.md`
  2. 必须先返工本章正文
  3. 如返工后仍失败，再回查 `03-chapters.md` 的章纲可写性证明与场景级字数预算

- 若连续 **2 章** 未达成稿标准：
  1. 暂停当前批次
  2. 先回查本批章纲是否存在“天然摘要章”
  3. 必要时先合并章节或补场景，再继续往下写

- 若连续 **2 章** 都属于“场景不足失败”：
  1. 默认判定为 `03` 章纲层预算失真
  2. 不应继续只在 `04` 里硬拉字数
  3. 必须回到 `03` 重做该批章纲预算

#### C.5 体量偏高复查（新增软约束）

如果本章不是摘要稿，也不是场景不足失败，但正文字数明显高于目标区间，则不要直接判失败，先做**润色压缩复查**。

建议分层处理：

- **2000~2500 字**：理想区间
- **2500~3000 字**：可接受浮动区间
  - 重点检查是否存在明显重复解释、重复心理或信息复述
- **3000 字以上**：进入润色压缩流程

当正文达到 **3000+** 时，优先动作不是回查章纲，而是先删减以下内容：

1. 重复表达同一判断的话
2. 作用重复的心理描写
3. 不推进事件的信息复述
4. 纯气氛、纯感慨、纯总结段

不应优先删减的内容：

1. 关键对话回合
2. 关系变化节点
3. 本章兑现点
4. 章末钩子
5. 下一章必须承接的状态变化

压缩后的处理顺序：

1. 先保结构
2. 再压废话
3. 压缩后若回到合理区间，可正常放行
4. 若压缩后仍明显超长，再回查 `03` 是否存在单章装载过满

#### C.6 证据公开代价检查（新增硬约束）

如果本章出现“主角拿到关键证据 / 证人 / 把柄 / 文书 / 实物残留”的情节，正文必须检查证据是否能立刻公开。

至少回答：

1. 公开证据会牵连谁
2. 公开证据会暴露哪条隐藏线、证人线或资源线
3. 对手会怎样反扑、销毁证据或反咬证据来源
4. 主角为什么选择立刻公开、暂缓公开、换方式公开，或只公开一部分

禁止写成：

1. 证据一出现就自动翻盘
2. 证据来源没有任何风险
3. 证人和帮手被公开后没有代价
4. 主角只靠一句判断就绕过正式流程或现实压力

如果本章章纲要求“不能立刻翻盘”“隐藏证人不能公开”“证据来源有风险”，但正文没有写出公开代价，应判为**越界检查失败**，不能进入状态回填。

### 3. 风格要求

- 以正文呈现，不写提纲腔
- 保持番茄网文节奏，段落不要过长
- 每一小段都应推动：事件 / 情绪 / 信息，至少占一项
- 如果是爽点章，爽点必须落地，不能只铺垫不兑现
- 如果是施压章，压力必须具体，不能抽象空转
- 信息进入要快，不要长时间空转铺垫
- 专业感优先用动作、对话、对峙和现场判断来落，不要只靠主角站着讲解
- 章末必须给读者明确的继续阅读理由

### 3.5 首章特殊要求

如果这是第1章，额外满足：

- 在较短篇幅内给出主角是谁
- 尽快说明主角当前现状
- 尽快抛出异常、危险或强冲突
- 尽快给出“这个故事为什么值得继续看”的钩子
- 在前半章内尽量同时完成：主角是谁、现在有多惨、眼前麻烦是什么、主角与别人不一样的动作或判断是什么

禁止把首章写成：

- 背景说明书
- 大段设定介绍
- 没有冲突的日常流水账
- 只有氛围没有事件
- 前半章主要内容只是观察环境、回忆背景、总结处境

### 4. 视角要求

- 默认使用本章指定视角
- 如必须切视角，只能服务于本章核心事件
- 视角切换不能破坏阅读流畅性

### 5. 章末要求

章末钩子至少满足其一：
- 新危机出现
- 旧危机升级
- 新信息只揭开一半
- 人物刚做出关键决定

同时满足：
- 下一章要发生什么，读者必须能大致看出来
- 禁止只用“他隐约觉得不对劲”“事情没有这么简单”这类模糊句子充当钩子

### 6. 质量底线

- **去 AI 味**：避免空泛总结、套路化转折提示、过度解释人物情绪、像在复述提纲
- **合法合规**：不得输出违规内容，不做不良引导
- **边界清晰**：可以玩热门梗和轻松吐槽，但不能靠明显冒犯、人身攻击、恶俗羞辱制造效果
- **必须原创**：严禁抄袭，严禁直接模仿具体作品的桥段、句式、对白、结构
- **番茄向不是模仿某本书**：只学节奏和信息推进效率，不学具体内容

### 7. 连续长跑补充检查

如果这一章属于整卷连续测试中的一部分，额外检查：

- 本章是否重复使用了前 2~3 章已经用过的同类兑现方式
- 本章是否继续把某个配角写成功能型工具人
- 本章是否让长期对手只会重复同一种施压动作
- 本章是否出现“上一章刚起的新线，这一章完全失踪”的断线
- 本章是否为下一阶段埋入足够可持续的问题，而不只是解决眼前问题
- 如果连续 5 章都由主角稳定解题、没有真实失手，视为中段风险
- 如果连续 3 章都靠同类围观、同类震惊、同类嘴压人来推动，视为卖点疲劳风险
- 如果连续 2 章写完后都更像“摘要稿”而非正文章节，视为流程故障，必须暂停批次回查章纲
- 如果连续 2 章都不是摘要稿、但明显写不满成稿体量，视为“场景预算失败”，必须回查 `03`
```

## 使用建议

1. 先生成正文，再看自检，不要反过来写成说明文。
2. 如果自检里出现“结束状态不一致”，优先改正文，不先改细纲。
3. 如果自检里“成稿判定”不通过，本章禁止进入 `05-update-state.md`。
4. 如果连续 2 章都出现“越界”问题，说明前面的细纲或交接卡过松，需要回查 `03-chapters.md` 的输出。
5. 如果连续 2 章未达成稿标准，先暂停后续批次正文，回查章纲，再继续往后写。
6. 如果失败类型是“场景不足失败”，优先回查 `03` 的场景级字数预算，而不是继续硬扩写。
7. 如果本章实际体量明显落在 800~1400 字区间，且原因已确认是场景不足，则本章**不得**勾选“达到可进入状态回填的最低成稿标准”。
8. 如果连续 3~5 章都出现类似问题，先暂停后续批次正文，更新工作流并回归测试前面关键章，再继续往后写。
9. 如果本章超过 3000 字，先做一轮润色压缩，再决定是否需要回查 `03`，不要直接按失败处理。
10. 如果本章拿到关键证据后直接翻盘，先检查“证据公开代价”；若公开会暴露隐藏证人、证据来源或重要资源线，正文必须改为“暂缓使用 / 部分使用 / 换方式使用 / 被迫承担代价”。
