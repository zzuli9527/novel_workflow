# Phase 4：章节细纲 → 单章正文

> 使用方式：一次只写一章。正文生成后先执行机械检查和本章自检，通过后才能进入 `05-update-state.md`。


## 1. 正文目标

- 写成可直接阅读的正文章节，不写摘要稿。
- 从交接卡指定状态启动。
- 完成本章必做结果。
- 不触碰禁止结果。
- 让修仙主线发生真实变化。
- 让搞笑来自人物、规则和行动。
- 保留细纲要求的代价或后果。
- 以具体行动方向结束。

## 2. 场景执行

### 每个场景需要

- 明确地点和参与角色
- 角色当前目标
- 可见动作
- 至少一轮有效回应或对抗
- 信息、关系、资源或能力变化
- 场景结果
- 进入下一场景的原因

不要用概括句快速跳过细纲中的核心场景。

如果正文主要由以下内容组成，应判为摘要化：

- “随后”“之后”“接下来”连续概括多件事
- 一段话完成原本应展开的对话或冲突
- 旁白直接说明人物关系变化
- 只写结果，不写角色如何行动
- 只复述章纲中的节点

## 3. 修仙一致性

正文必须遵守：

- 当前境界和能力边界
- 已知功法、法术和法宝规则
- 当前伤势和限制
- 当前资源数量
- 当前世界规则
- 已确认的突破条件

禁止：

- 临时创造新技能救场
- 无前置条件突破
- 使用已经耗尽或丢失的资源
- 忽略上一章伤势
- 同一能力突然扩大效果
- 只用一句“顿悟”跳过重大成长过程

如果本章是突破或修炼试错章，必须写出：

- 准备
- 尝试
- 暴露的问题
- 调整
- 结果
- 新边界或代价

## 4. 搞笑执行

### 正确方式

- 角色认真追求目标，行为之间产生反差。
- 不同角色基于各自信息作出合理判断。
- 世界规则被认真执行后产生意外结果。
- 笑点通过动作、对话、停顿和后果体现。
- 回调在新语境中产生新效果。

### 禁止方式

- 全员为笑点同时失去判断力。
- 旁白告诉读者某件事很好笑。
- 连续堆网络梗、谐音梗和流行语。
- 所有角色都用同一种吐槽语气。
- 每个场景都靠围观、震惊和沉默结算。
- 重复上一章同一误会，没有新增后果。
- 用搞笑取消伤势、损失、恐惧和关系破裂。

### 喜剧密度

按项目喜剧圣经执行。

高压、受伤、离别或重大失败场景允许降低笑点密度。降低密度不等于角色突然失去原有语言和性格。

## 5. 语言和节奏

- 段落适合移动端阅读，但不要机械切成一句一段。
- 对话必须有说话目的和立场变化。
- 修仙设定通过行动、选择和结果体现。
- 不连续使用作者总结句替读者下结论。
- 不反复解释已经清楚的信息。
- 不直接模仿具体作品的句式、桥段和人物。
- 现代词汇和网络表达是否可用，以项目喜剧圣经为准。

## 6. 开章要求

### 普通章节

前段尽快承接上一章动作或结果，不重复大段回顾。

### 第 1 章

尽快给出：

- 主角
- 当前处境
- 一个修仙世界规则
- 眼前问题
- 主角的行动差异
- 本书主要喜剧气质

首章不能从大段世界历史、境界说明或无冲突日常开始。

## 7. 章末要求

章末必须出现一个具体的继续阅读理由：

- 角色开始新的行动
- 当前条件发生变化
- 新信息迫使角色重新选择
- 代价开始兑现
- 资源或能力出现新限制
- 误会进入新的阶段
- 某个角色作出明确决定

不能只写：

- “事情没有那么简单”
- “他隐约觉得不对”
- “真正的麻烦才刚开始”

除非同时给出麻烦的具体载体或下一步动作。

## 8. 机械长度闸门

实际长度由代码统计，模型自报数字不能作为放行依据。

默认统计：

```text
只统计正文
排除标题、自检、状态表、作者说明和分隔线
去除空白字符后统计字符数
```

默认判定：

| 实际长度 | 结论 | 后续动作 |
| ---: | --- | --- |
| 0～1799 | 严重不足 | 禁止回填；重写或回查章纲 |
| 1800～1999 | 轻微不足 | 禁止回填；按场景缺口扩写 |
| 2000～3000 | 通过 | 继续其他检查 |
| 3001～3500 | 可接受浮动 | 检查重复与冗余 |
| 3501 以上 | 超长复查 | 压缩后重新检查 |

如果运行配置使用其他范围，以运行配置为准。

## 9. 场景不足失败

出现以下情况时，不允许靠增加解释硬补长度：

- 只有一个短场景。
- 核心冲突被一句话解决。
- 去掉心理和设定说明后，动作链极短。
- 继续扩写只能重复笑点或反应。
- 场景之间没有因果。

处理顺序：

```text
回查场景预算
→ 补充有功能的场景
→ 或与相邻章节合并
→ 重新生成正文
```

## 10. 定向扩写规则

轻微不足时，可以扩写低于预算的场景。

允许增加：

- 动作和回应回合
- 现场条件变化
- 资源使用过程
- 修炼尝试过程
- 关系反应
- 喜剧行动连锁
- 后果落地

禁止增加：

- 下一章剧情
- 新能力、新法宝或新资源
- 重复心理
- 重复笑话
- 大段设定解释
- 改变本章结局

同一章最多自动扩写两次。仍不通过时，退回章纲。

## 11. 连续章节检查

额外检查最近章节：

- 是否重复同一主要喜剧机制
- 是否重复同一种围观反应
- 是否让同一配角连续承担工具人功能
- 是否连续只有搞笑，没有修仙推进
- 是否连续只有修炼说明，没有喜剧兑现
- 是否丢失上一章新增限制或资源
- 是否让重大代价下一章立即消失
- 是否让误会在没有新后果时继续拖延

连续两章出现相同结构性失败时，暂停批次并回查 `03-chapters.md`。

## 进入 Phase 5 的条件

只有同时满足以下条件才能进入状态回填：

- 机械长度闸门通过。
- 正文不是摘要稿。
- 必做结果完成。
- 没有出现禁止结果。
- 没有修炼体系硬矛盾。
- 搞笑没有依赖全员降智或取消严肃后果。
- 资源变化与当前余额一致。
- 知识和误会按角色区分且没有互相矛盾。
- 核心角色没有退化为相同声音的工具人。
- 多条剧情线不是互不相关的并排摘要。
- 章末状态可用于下一章。
