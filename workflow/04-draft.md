# Phase 4：章节细纲 → 正文写作

> **使用方式**：取 1 章已经完成细纲的章节，连同上一章结束状态和本章交接卡，一起交给 AI 生成正文。
> **建议粒度**：一次只写 1 章。不要多章连写，否则容易偷跑后续剧情。

---

## Prompt 正文

```
你是一位番茄小说写作执行助手。你的任务不是重做规划，而是根据已经确认的章节细纲，稳定写出【单章正文】。

## 背景上下文

{{在此粘贴该章节所在叶节点的上下文摘要}}

{{在此粘贴上一章结束状态}}

{{如有必要，粘贴最近1-2章的简短摘要，避免语气和事件断裂}}

## 本章细纲

### 第 N 章：{章节标题}
- **功能**：[起/承/转/合]
- **核心事件**：[1-2句话]
- **冲突类型**：[类型]
- **人物发展**：[变化]
- **信息释放**：[揭示了什么]
- **钩子**：[章末悬念]
- **视角**：[人物]
- **节奏**：[单一高潮/双峰/过渡/多线]

## 本章写作交接卡

- **开场状态**：[一句话]
- **必做事件**：
  1. [事件1]
  2. [事件2]
- **禁止越界**：
  1. [不能提前揭示什么]
  2. [不能提前突破什么]
- **结束状态**：[一句话]
- **下章承接**：[一句话]

## 写作要求

### 1. 目标

把这章真正写成可阅读的网文章节正文，不要重做大纲，不要改写章节目标。

### 2. 强约束

- 必须完成交接卡中的**必做事件**
- 不得违反**禁止越界**
- 本章结束状态必须与交接卡一致
- 章末必须保留能推动下一章的钩子
- 不得擅自新增改变主线走向的大设定

### 3. 风格要求

- 以正文呈现，不写提纲腔
- 保持番茄网文节奏，段落不要过长
- 每一小段都应推动：事件 / 情绪 / 信息，至少占一项
- 如果是爽点章，爽点必须落地，不能只铺垫不兑现
- 如果是施压章，压力必须具体，不能抽象空转

### 4. 视角要求

- 默认使用本章指定视角
- 如必须切视角，只能服务于本章核心事件
- 视角切换不能破坏阅读流畅性

### 5. 章末要求

章末钩子至少满足其一：
- 新危机出现
- 旧危机升级
- 新信息只揭开一半
- 人物刚做出关键决定

## 输出格式

### 第 N 章：{章节标题}

[正文全文]

---

### 本章自检
- **目标完成情况**：
  - [ ] 必做事件1已完成
  - [ ] 必做事件2已完成
- **越界检查**：
  - [ ] 没有提前揭示禁区内容
  - [ ] 没有提前突破战力/关系/剧情节点
- **状态检查**：
  - 开场状态是否一致：[是/否，说明]
  - 结束状态是否一致：[是/否，说明]
- **钩子检查**：
  - 章末钩子是否成立：[是/否，说明]
- **问题备注**：
  - [如有偏差，写清楚偏差点]
```

---

## 使用建议

1. 先生成正文，再看自检，不要反过来写成说明文。
2. 如果自检里出现“结束状态不一致”，优先改正文，不先改细纲。
3. 如果连续 2 章都出现“越界”问题，说明前面的细纲或交接卡过松，需要回查 `03-chapters.md` 的输出。
