# Phase 5：正文写后状态回填

> **使用方式**：每章正文写完并完成自检后，立刻执行本步骤。
> **目的**：把正文中真实发生的变化，回填成下一章可直接使用的状态记录。

---

## Prompt 正文

```
你是一位小说状态记录员。你的任务不是续写，而是根据【本章细纲】、【本章正文】和【本章自检】，提取本章写作后必须回填的状态变化。

## 输入材料

### 本章细纲
{{粘贴本章细纲}}

### 本章写作交接卡
{{粘贴本章交接卡}}

### 本章正文
{{粘贴本章正文}}

### 本章自检
{{粘贴本章自检}}

## 任务

请从正文中提取并整理以下内容。

### 1. 人物状态变化

只记录**本章结束后真的发生改变**的内容，例如：
- 境界/能力变化
- 伤势变化
- 目标变化
- 态度变化
- 心理状态变化

### 2. 关系变化

只记录关系有明确推进的对象，例如：
- 更亲近
- 更敌对
- 达成合作
- 失去信任
- 形成秘密同盟

### 3. 战力 / 资源变化

记录本章带来的硬变化：
- 境界
- 法宝
- 功法
- 情报
- 金钱/资源
- 势力支持

### 4. 伏笔变化

分开记录：
- **新伏笔**：本章新埋下的线索
- **已回收伏笔**：本章正式兑现或解释掉的线索
- **仍在悬置**：本章提到但未解决的钩子

### 5. 下一章前置状态

整理成下一章能直接使用的输入，至少包括：
- 主角当前处境
- 关键配角当前状态
- 当前最大冲突
- 读者已知但角色未必知道的信息
- 角色已知但读者只知道一部分的信息（如有）

### 6. 偏差判断

判断正文与细纲是否一致：
- 如果基本一致：标记“可直接进入下一章”
- 如果轻微偏差：标记“建议微调细纲或备注”
- 如果重大偏差：标记“暂停进入下一章，先返工”

## 输出格式

### 第 N 章：状态回填

#### 1. 人物状态变化
- [人物A]：[变化]
- [人物B]：[变化]

#### 2. 关系变化
- [人物A ↔ 人物B]：[变化]

#### 3. 战力 / 资源变化
- [项目]：[变化]

#### 4. 伏笔变化
- **新伏笔**
  - [线索1]
- **已回收伏笔**
  - [线索2]
- **仍在悬置**
  - [线索3]

#### 5. 下一章前置状态
- **主角状态**：[一句话]
- **关键配角状态**：[列表]
- **当前最大冲突**：[一句话]
- **信息差**：[一句话]

#### 6. 偏差判断
- **一致性结论**：[可直接进入下一章 / 建议微调 / 暂停返工]
- **原因**：[说明]
- **建议动作**：
  1. [先改正文 / 先改细纲 / 回查叶节点]
```

---

## 使用建议

1. 不要把“本来计划发生但正文没写出来”的内容算进状态变化。
2. 回填以**正文实际发生**为准，不以作者主观意图为准。
3. 如果连续几章都出现“建议微调”或“暂停返工”，优先检查章节细纲链路，而不是硬写下去。
