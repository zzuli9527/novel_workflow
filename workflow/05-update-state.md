# Phase 5：正文写后状态回填

> **使用方式**：每章正文写完并完成自检后，立刻执行本步骤。
> **目的**：把正文中真实发生的变化，回填成下一章可直接使用的状态记录。

> **整卷实跑补充**：如果目标是第一卷完整测试，除了逐章回填，还要维护批次级状态账本。每完成 5~10 章，额外汇总长期人物、长期对手、长期线索和阶段问题，作为下一批次细纲与正文的统一输入。

---

## Prompt 正文

```
你是一位小说状态记录员。你的任务不是续写，而是根据【本章细纲】、【本章正文】和【本章自检】，提取本章写作后必须回填的状态变化。

## 输入材料

### 本章细纲
{{粘贴本章细纲}}

### 本章写作交接卡
{{粘贴本章交接卡}}

### 本章正文
{{粘贴本章正文}}

### 本章自检
{{粘贴本章自检}}

## 任务

请从正文中提取并整理以下内容。

### 0. 正文准入检查（新增硬约束）

在回填状态之前，先判断当前输入的“正文”是否具备进入状态回填的资格。

只有满足以下条件，才能继续执行状态回填：

1. 本章文本是**正文章节**，不是摘要、提纲、复盘或场景概述
2. 本章已完成 `04-draft.md` 中的本章自检
3. 本章自检里的**成稿判定**为通过
4. 本章确实展开了交接卡要求的主要事件，而不是只在总结里提到

如果不满足：

- 直接输出：`禁止回填，先返工正文`
- 不要继续提取人物、关系、资源、伏笔
- 明确说明：是字数问题、摘要问题、场景缺失问题，还是自检缺失问题

### 1. 人物状态变化

只记录**本章结束后真的发生改变**的内容，例如：
- 境界/能力变化
- 伤势变化
- 目标变化
- 态度变化
- 心理状态变化

### 2. 关系变化

只记录关系有明确推进的对象，例如：
- 更亲近
- 更敌对
- 达成合作
- 失去信任
- 形成秘密同盟

### 3. 战力 / 资源变化

记录本章带来的硬变化：
- 境界
- 法宝
- 功法
- 情报
- 金钱/资源
- 势力支持

### 4. 伏笔变化

分开记录：
- **新伏笔**：本章新埋下的线索
- **已回收伏笔**：本章正式兑现或解释掉的线索
- **仍在悬置**：本章提到但未解决的钩子

### 5. 对手认知 / 误判状态（新增硬约束）

如果本章存在试探、误导、假线、半真半假的证据、对手反查等情节，必须记录对手当前认知。

至少包括：

- 对手已经知道什么
- 对手还不知道什么
- 对手误以为什么
- 对手正在查什么
- 主角是否故意制造了假线或信息差

如果这类信息没有回填，下一章很容易写丢对手行动动机，应标记为“建议补充状态”。

### 6. 下一章前置状态

整理成下一章能直接使用的输入，至少包括：
- 主角当前处境
- 关键配角当前状态
- 当前最大冲突
- 读者已知但角色未必知道的信息
- 角色已知但读者只知道一部分的信息（如有）

### 7. 偏差判断

判断正文与细纲是否一致：
- 如果基本一致：标记“可直接进入下一章”
- 如果轻微偏差：标记“建议微调细纲或备注”
- 如果重大偏差：标记“暂停进入下一章，先返工”

### 8. 批次级账本（整卷实跑时必须补）

每 5~10 章额外整理：

- **长期人物状态表**：主角、家人、核心配角、长期对手
- **长期关系表**：盟友 / 对手 / 利用关系 / 潜在断裂关系
- **长期资源表**：金钱、名声、资格、渠道、关键证据
- **长期线索表**：已埋、已回收、必须继续追的线
- **对手认知账本**：对手已知、未知、误判、正在查、主角制造的信息差
- **疲劳风险表**：哪些卖点、冲突、配角反应已经开始重复
- **回归测试输入**：如果要重写前面某章，必须保留哪些后续既有结果

## 输出格式

### 第 N 章：状态回填

#### 1. 人物状态变化
- [人物A]：[变化]
- [人物B]：[变化]

#### 2. 关系变化
- [人物A ↔ 人物B]：[变化]

#### 3. 战力 / 资源变化
- [项目]：[变化]

#### 4. 伏笔变化
- **新伏笔**
  - [线索1]
- **已回收伏笔**
  - [线索2]
- **仍在悬置**
  - [线索3]

#### 5. 对手认知 / 误判状态
- **对手已知**：[一句话]
- **对手未知**：[一句话]
- **对手误判**：[一句话，如无则写“无明显误判”]
- **对手正在查**：[一句话]
- **主角制造的信息差**：[一句话，如无则写“无”]

#### 6. 下一章前置状态
- **主角状态**：[一句话]
- **关键配角状态**：[列表]
- **当前最大冲突**：[一句话]
- **信息差**：[一句话]

#### 7. 偏差判断
- **一致性结论**：[可直接进入下一章 / 建议微调 / 暂停返工]
- **原因**：[说明]
- **建议动作**：
  1. [先改正文 / 先改细纲 / 回查叶节点]
```

---

## 使用建议

1. 不要把“本来计划发生但正文没写出来”的内容算进状态变化。
2. 回填以**正文实际发生**为准，不以作者主观意图为准。
3. 如果当前文本根本不是成稿级正文，禁止为了流程推进而强行回填。
4. 如果连续几章都出现“建议微调”或“暂停返工”，优先检查章节细纲链路，而不是硬写下去。
5. 整卷测试时，批次账本比单章回填更重要；下一批细纲与正文应优先读取批次账本，而不是只看上一章。
6. 如果本章使用假线、诱导、半真证据或信息差，对手认知必须写清楚；否则下一章不得直接默认对手按作者意图行动。
7. 如果连续 5~10 章都存在对手误判或假线诱导，批次级账本必须汇总“对手认知账本”，否则暂停下一批细纲生成，先补账本。
