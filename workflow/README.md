# 番茄小说细纲工作流

> 从想法到全书章节细纲的递归生成系统。
>
> **核心思想**：故事点驱动 × 模块化叙事 × 多线并行 × 递归展开 × 战力天花板锁死。

---

## 核心理念

传统做法：1000章 → 均分 10×100 → 再均分 10×10 → 填内容。**问题**：按章节拆分，叙事节奏被机械切割；战力随着地图扩大不断膨胀；"起"太多导致故事跳跃。

本工作流做法：

```
用户想法（只有起点）
    │
    ├─ Phase 0-A: 终点推演 —— 从起点推导3个可能的终点，选最优
    ├─ Phase 0-B: 战力天花板 —— 锁死最高境界、进度映射、战力锚点
    ├─ Phase 0-C: 顶层框架 —— 给定起终点 + 模块库 → 顶层故事点树
    │
    ▼
起点 A ─────── 递归插入故事节点 ───────→ 终点 Z
         │                               │
         每次插入从模块库(1-5)选模块展开
         模块决定节拍序列（压争破得组合）
         展开到"不可再分的故事事件"→ 叶节点
         最后才把叶节点映射到章节
```

- **拆分对象**：故事点（叙事节拍），不是章节
- **拆分方式**：模块驱动递归，不是机械等分
- **章节数量**：故事点映射后的**结果**
- **多线处理**：每条线独立展开，用模块4（多线并拢）交汇
- **战力控制**：天花板在规划阶段锁死，不从纵向突破

## 模块库速查

| 模块 | 名称 | 序列 | 用途 |
|------|------|------|------|
| 1 | 标准打脸 | 压→争→破→得 | 通用打脸闭环，最常用 |
| 2 | 螺旋施压 | 压→争→压→争→破 | 压力叠加，不给奖励。**后必须接模块3** |
| 3 | 集中收割 | 破→得→得→得 | 纯释放。**仅在模块2之后使用** |
| 4 | 多线并拢 | 压(A)→压(B)→争(交汇)→破 | 两条叙事线碰撞 |
| 5 | 无缝钩子 | 得→压 | 结算旧账同时开新局，过渡用 |

详细定义见 `modules.md`。

## 文件说明

| 文件 | 用途 | 执行次数 |
|------|------|----------|
| `modules.md` | 叙事模块库（模块1-5 + 扩展位） | 被所有 Phase 引用 |
| `00-global.md` | Phase 0-A/B/C：终点推导 + 战力天花板 + 顶层框架 | **1 次** |
| `01-expand.md` | Phase 1：故事点递归展开 | **N 次**（每个非叶节点 1 次） |
| `02-merge.md` | Phase 2：多线交汇处理 | **按需**（每个复杂交汇点 1 次） |
| `03-chapters.md` | Phase 3：章节映射 + 章级细纲 | **M 次**（批量处理叶节点） |

## 使用步骤

### Step 0：阅读模块库

先读 `modules.md`，理解五个模块的节拍序列和适用场景。这是整个工作流的"语法"。

### Step 1：运行 Phase 0

打开 `00-global.md`，按顺序执行三个子步骤：
1. **0-A 终点推演**：把你的起点创意输入，AI 给 3 个终点方向，选一个
2. **0-B 战力天花板**：锁定最高境界、进度映射、锚点人物、禁止项
3. **0-C 顶层框架**：综合起点+终点+战力，AI 产出顶层故事点树

### Step 2：运行 Phase 1（循环）

对 Phase 0 产出中的**每个非叶节点**：
1. 打开 `01-expand.md`，填入节点信息
2. AI 按模块节拍序列展开
3. 子节点中的非叶节点 → 继续 Phase 1
4. 直到全部为叶节点

建议：**深度优先**，先完整展开一条分支。

### Step 3：运行 Phase 2（按需）

对标记为交汇点的叶节点，打开 `02-merge.md` 细化交叉节奏。

### Step 4：运行 Phase 3

分批处理叶节点：
1. 打开 `03-chapters.md`
2. 每次输入 1-3 个连续叶节点（约 5-20 章）
3. AI 产出逐章细纲
4. 手动维护章号计数器

## 关键约束速查

| 约束 | 说明 |
|------|------|
| ⚠️ 模块2→3 | 螺旋施压后必须接集中收割，否则读者弃书 |
| ⚠️ 压必有争 | 任何"压"之后必须在同一模块内看到"争" |
| ⚠️ 压不连三 | 同一模块内不出现连续3个同类型节拍 |
| ⚠️ 战力天花板 | 全书最高境界一旦设定，不出现更高境界 |
| ⚠️ 故事点≠章节 | 展开时不分配章节号，那是 Phase 3 的事 |
| ⚠️ 每章有钩子 | Phase 3 输出中，每章钩子不为空 |

## 扩展

模块库中的 E/F 位可用于添加你自己的叙事模块。当你在创作中发现新的有效节拍模式时，将其抽象为模块加入 `modules.md`。
