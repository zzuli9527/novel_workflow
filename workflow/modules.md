# 叙事模块库（番茄小说版）

> 番茄读者在手机上碎片化阅读。每一章结束都可能弃书。
> 所以模块设计原则：**短闭环、快反馈、密钩子**。

---

## 节拍类型

旧版"起承转合"问题：起（setup）太多时故事跳跃、缺乏抓手。

新版用四个**动作型节拍**，每个节拍推动具体事件而非抽象叙事功能：

| 节拍 | 含义 | 读者情绪 | 弃书风险 |
|------|------|----------|----------|
| **压** | 建立压力——不公平、威胁、轻视、危机降临。明确"对手是谁"、"代价是什么"。 | 愤怒/不平/紧张 | ⚠️ 如果压而不争，读者会憋屈弃书 |
| **争** | 主角抗争——正面迎击、暗中布局、收集资源、寻找破绽。让读者看到"翻盘的希望"。 | 期待/紧张 | ⚠️ 如果争而无进展（连续吃瘪但不积累），弃 |
| **破** | 突破翻盘——底牌亮出、局势逆转、对手被打脸、秘密揭晓。情绪的爆发点。 | 爽/震撼/释放 | ✅ 安全区 |
| **得** | 收获结算——战利品、地位提升、关系确认、新信息获得。情绪释放+自然引出下一轮。 | 满足/好奇 | ✅ 安全区，但得太多而无新压会无聊 |

**核心约束**：
- "压"之后必须在同一模块内看到"争"——不能让读者纯憋屈
- "破"之前必须有至少一个"压"积累——没有压力的翻盘是自嗨
- 一个模块内不能出现连续3个同类型节拍——情绪单调

## 模块 1：标准打脸

```
压 → 争 → 破 → 得
```

**算法**：一个完整的打脸闭环。读者在"压"中建立对反派的愤怒，在"争"中看到主角的反抗，在"破"中获得释放，在"得"中感到满足。

**节拍时长参考**（以该模块覆盖的故事点为单位）：
- 压：20%
- 争：35%
- 破：25%
- 得：20%

**情绪曲线**：
```
    愤怒          紧张期待      爽！        满足
     │            ┌─争─┐     ┌破┐      ┌得┐
     │   ┌压┐    │    │    │  │     │  │
     └───┘  └────┘    └────┘  └─────┘  └──
```

**适用场景**：最通用的模块，适合任何需要"先抑后扬"的故事点。可嵌套（大模块的"争"内部可以嵌套一个小模块1）。

**番茄适配**：打脸是番茄的核心爽点。这个模块确保每轮打脸都有完整的情绪闭环。

**示例**：
- 压：族会上主家子弟当众嘲讽苏尘是"废材父子"，要求逐出家族
- 争：苏尘提出三月族比之约，暗中在矿脉苦修
- 破：族比日连败三大天才，全场死寂
- 得：老祖收徒，资源到位，主家势力清洗

---

## 模块 2：螺旋施压

```
压 → 争 → 压 → 争 → 破
```

**算法**：第一个压力尚未完全解决，第二个更强的压力叠加进来。主角的"争"不断遇到新的阻碍，压力螺旋式上升。**刻意不给"得"**——让读者的期待持续累积。

**情绪曲线**：
```
紧张度
    │           压₂
    │    争₁  ┌─┘
    │   ┌┘    │  争₂     ┌破┐
    │ 压₁     └──┘      │  │
    │ │                 └──┘
    └────────────────────────
         （没有得——奖励被延迟）
```

**适用场景**：
- 主角低谷期/被全面打压
- 大战前夕的多方围剿
- 多股势力同时发难

**⚠️ 铁律**：模块2之后**必须**接模块3（集中收割）。压抑不释放 = 读者弃书。这里的"破"只是打开了翻盘的缺口，还没有收获。

**示例**：
- 压₁：苏尘被栽赃勾结魔道，面临宗门审判
- 争₁：苏尘收集证据自证清白
- 压₂：旧仇人买通执法长老，证据被销毁，追加"毁尸灭迹"罪名
- 争₂：苏念尘（7岁）偷学禁术重新收集证据，但被宗主发现
- 破：审判日，苏念尘当众播放留影石，长老通敌铁证公开

---

## 模块 3：集中收割

```
破 → 得 → 得 → 得
```

**算法**：接在模块2之后的纯释放。没有新的压力，只有连续的收获。每一个"得"对应模块2中读者积累的一个期待。

**情绪曲线**：
```
爽感
    │      得₁   得₂   得₃
    │     /  \  /  \  / 
    │  破      \/    \/
    │  │
    └──┴──────────────────
```

**适用场景**：
- **紧接在模块2之后**（唯一触发条件）
- 阶段性Boss战胜利后的缴获
- 翻盘后的地位跃升、人脉扩展、势力建立

**⚠️ 不能独立使用**：没有模块2积累的压力，模块3的三个"得"就是空刷数据，读者无感。

**示例**（接上面模块2）：
- 破：留影石公开，执法长老被拿下
- 得₁：苏尘洗清罪名，晋升为杂役峰峰主
- 得₂：苏尘获得独立执法权，开始推行"苏氏修炼法"
- 得₃：废材联盟第一批突破，苏尘名声传遍天剑宗周边

## 模块 4：多线并拢

```
压(A) → 压(B) → 争(交汇) → 破
```

**算法**：两条叙事线各自建立压力，然后在"争"阶段自然交汇，在"破"阶段集中引爆。交汇点必须是事件驱动的——两条线因同一个事件被迫碰撞，而非生硬切换视角。

**与旧版模块A的关键区别**：
- 旧版：起-起-起-承-转-合（三线并拢，6拍，太多起导致跳跃）
- 新版：最多两线同时铺，且铺完立刻交汇，不会让读者等太久

**情绪曲线**：
```
线A：  压(A)────────┐
                     ├─争(交汇)──破──
线B：       压(B)───┘

读者：  "这条线怎么了" → "这也出事了？" → "它们要撞上了！" → "爆了！"
```

**适用场景**：
- 两条独立叙事线需要交汇时
- 主角线和反派线同时推向碰撞
- 两条看似无关的线索（比如苏尘的宗门危机 + 苏念尘的禁术发现）最终指向同一件事

**约束**：如果涉及3条以上的线，先两两并拢，再用一次模块4并拢结果。不要一次并3条。

**示例**：
- 压(A)：苏尘在矿脉深处发现上古洞府，但主家子弟带人堵住了出口
- 压(B)：苏念尘在族中被圣地使者强行测灵根，即将暴露天灵根
- 争(交汇)：上古洞府中苏尘触发的禁制与族中测灵台的阵法产生共鸣——两处阵法是同一人所留
- 破：洞府传承认主苏尘，测灵台炸裂，圣地使者被震退，父子暗中完成第一次联手

---

## 模块 5：无缝钩子

```
得 → 压
```

**算法**：最短的模块（可短至1-3章）。在上一段故事的"得"正在兑现的同时，新一段故事的"压"已经入侵。读者无法找到"停下来"的节点。

**情绪曲线**：
```
    满足感           好奇心（下一个是什么？）
    ┌得┐
    │  └──压（已经开始了）
    │      │
    └──────┘
    结束感被打破
```

**适用场景**：
- 两卷之间的过渡（上卷结算 + 下卷引子）
- 章节末尾的断章钩子
- 任何想让读者"再读一章"的位置

**注意**：模块5本身不产生完整的故事闭环，它只是粘合剂。不要连续使用模块5。

**示例**：
- 得：苏尘父子获得独立洞府，资源到位，全城来贺
- 压：贺礼中混入一封匿名信——"你父亲的死不是意外，天剑宗有人知道真相"

## 模块速查表

| 模块 | 序列 | 节拍数 | 触发条件 | 后续必须 |
|------|------|--------|----------|----------|
| 1 标准打脸 | 压→争→破→得 | 4 | 通用 | 任意 |
| 2 螺旋施压 | 压→争→压→争→破 | 5 | 通用 | **模块3** |
| 3 集中收割 | 破→得→得→得 | 4 | **仅在模块2之后** | 任意（推荐接模块5） |
| 4 多线并拢 | 压(A)→压(B)→争(交汇)→破 | 4 | 两条线需要交汇 | 任意 |
| 5 无缝钩子 | 得→压 | 2 | 章节/卷过渡 | 任意（推荐接模块1） |

## 典型组合链

```
开篇：    模块4（多线引入）→ 模块1（第一条打脸闭环）
发展：    模块1 → 模块1 → 模块2 → 模块3 → 模块5
高潮：    模块2 → 模块3 → 模块4（多线汇聚爆发）
转场：    模块5 → 模块1（新循环开始）
```
