# 修仙搞笑叙事模块库

> 使用方式：每个故事点先选择一个“修仙推进模块”，再选择一个主要“喜剧机制”。不要把固定节拍当成必须重复套用的剧情模板。

## 一、修仙推进模块

### P1：目标—阻碍—变招—结果—代价

```text
明确目标
→ 遇到具体阻碍
→ 使用已有能力或资源尝试
→ 根据现场变化调整方法
→ 得到阶段结果
→ 产生代价或后果
```

适合：大多数日常推进、任务、试炼、冲突和关系事件。

要求：

- 结果必须改变状态。
- 变招不能依赖临时出现的新能力。
- 代价可以是资源、暴露、关系、伤势、时间或新约束。

### P2：压力升级

```text
第一层压力
→ 初步应对
→ 第二层压力改变条件
→ 被迫重新选择
→ 得到不完整结果
```

适合：阶段中段、被追查、资源争夺、身份风险、多方压力。

要求：

- 第二层压力必须改变解决方式，不是单纯增加人数或强度。
- 结尾至少留下进展，不能纯吃瘪。
- 后续必须安排兑现，不得无限拖延。

### P3：修炼试错与突破

```text
当前瓶颈
→ 准备资源和方法
→ 第一次尝试暴露问题
→ 调整理解或执行方式
→ 成功、失败或半成功
→ 能力变化与代价落地
```

适合：修炼、炼丹、炼器、阵法、功法理解、能力开发。

要求：

- 突破条件来自已确认规则。
- 失败也必须产生信息或状态变化。
- 不能靠搞笑意外无代价完成重大突破。

### P4：探索与资源获取

```text
资源目标
→ 进入陌生环境
→ 发现规则或风险
→ 做出取舍
→ 获得资源或线索
→ 带出新的限制
```

适合：秘境、遗迹、坊市、任务地点、宗门区域和陌生势力。

要求：

- 新资源有来源、数量和使用限制。
- 探索必须改变认知或行动，不写成景点介绍。
- 得到资源后不能立刻忘记。

### P5：势力与关系博弈

```text
立场冲突
→ 条件交换或试探
→ 信息差改变判断
→ 角色作出选择
→ 关系重新定位
→ 后续义务或风险形成
```

适合：宗门、师徒、同门、家族、交易、合作和敌对关系。

要求：

- 各方有自己的利益，不只是等待主角表现。
- 关系变化必须有具体行为依据。
- 搞笑不能把重要角色写成一次性工具人。

### P6：阶段收束

```text
集中处理当前核心问题
→ 兑现前文积累
→ 给出可见结果
→ 结算资源、关系和能力
→ 保留未解决问题
→ 引出下一阶段行动
```

适合：一个 10～20 章故事单元的后段。

要求：

- 至少回收一项前文承诺。
- 阶段胜利不是全书问题彻底解决。
- 新问题必须从当前结果自然产生。

## 二、喜剧机制

### C1：性格反差

角色的目标、外在形象和真实反应之间形成反差。

有效方式：

- 严肃角色面对不擅长的日常问题。
- 胆小角色在关键处因特殊理由最先行动。
- 看似随意的角色对某条规则异常认真。

禁止：为了反差突然改变既有人格。

### C2：信息差与误会

不同角色掌握的信息不同，因此作出彼此合理、合在一起却滑稽的行动。

要求：

- 每个角色的判断都要符合自己已知信息。
- 误会每次推进必须增加新后果。
- 误会不能靠角色拒绝说一句正常的话无限拖延。
- 重要误会必须进入状态账本。

### C3：规则错位

角色认真执行世界规则，却产生意料之外的结果。

适合：宗门制度、任务规则、修炼限制、交易规则、礼法和身份规范。

要求：

- 规则必须提前存在。
- 结果不能凭空改变规则。
- 笑点之后仍要处理真实后果。

### C4：严肃场面失控

场面本来高度严肃，但一个合理变量使秩序逐步偏离。

要求：

- 失控由人物行动或既有规则引发。
- 角色仍然在认真解决问题。
- 不能把所有人写成只会吵闹。

### C5：行动连锁

一个为解决小问题的动作，引发连续且越来越难收拾的后果。

要求：

- 每一步都由上一步导致。
- 连锁至少推动一个主线状态。
- 结尾必须结算，不无止境堆事故。

### C6：回调升级

重新使用前文的人物习惯、物品、承诺、失败或笑点，但加入新语境和新后果。

要求：

- 不能原样重复。
- 本次回调必须增加信息、关系或后果。
- 使用后更新疲劳状态。

### C7：对话节奏

通过立场错位、答非所问、过度认真、停顿和话语权变化形成喜剧。

要求：

- 每个角色有独立语言习惯。
- 对话必须推动决定或关系。
- 不使用密集网络梗代替人物语言。

## 三、组合规则

每个故事点默认选择：

```text
1 个主要修仙推进模块
+
1 个主要喜剧机制
+
0～1 个轻量回调
```

高压或悲伤段落可以降低喜剧密度，但不能突然改变整本书的语言气质。

同一主要喜剧机制连续使用两章后，下一章应更换机制或显著改变使用对象和后果。

## 四、推荐组合

| 修仙模块 | 适合搭配的喜剧机制 | 注意事项 |
| --- | --- | --- |
| P1 日常推进 | C1 / C3 / C7 | 不要每次都靠嘴赢 |
| P2 压力升级 | C2 / C4 | 笑点不能取消压力 |
| P3 修炼试错 | C3 / C5 | 突破仍需真实条件 |
| P4 探索资源 | C2 / C5 / C6 | 新资源必须记账 |
| P5 关系博弈 | C1 / C2 / C7 | 各方不能降智 |
| P6 阶段收束 | C4 / C6 | 收束结果要清楚 |

## 五、疲劳检查

出现以下情况时，标记喜剧疲劳：

- 连续两章主要依赖同一种误会。
- 连续三章都有同类围观或震惊反应。
- 同一角色连续承担出丑和挨骂功能。
- 每次冲突都以嘴硬、打脸和群体沉默结束。
- 回调没有新增后果。
- 笑点越来越依赖旁白解释。

处理方式：

- 更换主要喜剧机制。
- 更换承担笑点的角色关系。
- 让笑点产生真实代价。
- 降低本章笑点密度，增加行动推进。
- 暂停使用已经疲劳的回调。
